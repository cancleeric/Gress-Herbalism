# 報告書 0130：跟猜決定面板 UI 重新設計

## 工作單編號
0130

## 完成日期
2026-01-26

## 完成內容摘要

根據 Stitch 設計稿，重新設計跟猜決定面板，延續中國風草藥主題設計風格。

### 主要變更

1. **UI 重新設計**
   - 新增中國風草藥紋理背景（fg-texture）
   - 新增角落裝飾圖案（fg-corner-tr、fg-corner-bl）
   - 更新標題區域：置中標題「跟猜階段」+ 紅色底線裝飾
   - 新增猜牌資訊區塊：顯示猜牌者名稱和猜測的顏色（pill 樣式）
   - 新增玩家決策狀態列表：卡片樣式垂直排列
   - 更新決定按鈕區：「不跟」和「跟猜」按鈕

2. **玩家狀態顯示**
   - 猜牌者：金色邊框 + 「猜牌者」標籤 + 「發起猜測」狀態
   - 已跟猜玩家：「跟猜」標籤（綠色）
   - 已不跟玩家：「不跟」標籤（灰色）
   - 當前決定者：橙色邊框高亮 + 「決定中...」標籤 + 放大效果
   - 等待中玩家：半透明 + 「等待中」文字（灰色斜體）

3. **等待狀態文字**
   - 猜牌者：「等待其他玩家按順序決定是否跟猜...」
   - 已決定玩家：「你選擇了「跟猜/不跟」，等待其他玩家...」
   - 未輪到玩家：「還沒輪到你，請等待...」

4. **顏色名稱中文化**
   - 使用 `COLOR_NAMES` 對照表顯示中文顏色名稱
   - 顏色標籤使用 pill 樣式配合對應顏色背景

5. **CSS 類名規範**
   - 所有新樣式使用 `fg-` 前綴避免衝突
   - 新增類名：
     - `.fg-modal` - 主容器
     - `.fg-texture` - 草藥紋理背景
     - `.fg-corner-tr`、`.fg-corner-bl` - 角落裝飾
     - `.fg-header`、`.fg-title`、`.fg-title-line` - 標題區
     - `.fg-guess-info` - 猜牌資訊區塊
     - `.fg-color-chip` - 顏色標籤
     - `.fg-players-section` - 玩家狀態區
     - `.fg-player-card` - 玩家卡片
     - `.fg-action-section` - 決定按鈕區
     - `.fg-btn-pass`、`.fg-btn-follow` - 按鈕樣式

## 遇到的問題與解決方案

1. **保留現有功能**
   - 問題：需要保留原有的跟猜邏輯
   - 解決：只修改 UI 結構，保留所有狀態判斷和事件處理函數

2. **玩家狀態判斷**
   - 問題：需要根據不同玩家狀態顯示不同樣式
   - 解決：使用條件渲染和動態 className 處理各種狀態

3. **響應式設計**
   - 問題：需要在手機和桌面都有良好顯示
   - 解決：按鈕區域使用 flex-col/sm:flex-row 切換佈局
