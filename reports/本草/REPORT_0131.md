# 報告書 0131：跟猜決定面板 UI 重新設計（修正版）

## 完成日期

## 完成內容摘要

根據 Stitch 設計稿，重新設計跟猜決定面板，延續中國風草藥主題設計風格。此工單修正 0130 的錯誤實作。

### 問題說明

工單 0130 實作時，新的跟猜面板被錯誤地添加到了第二個 return 語句（其他階段渲染部分），而非替換第一個 return 語句（playing 階段渲染部分）中的舊面板，導致遊戲中仍然顯示舊的 UI。

### 修正內容

1. **替換正確位置的面板**
   - 將第一個 return 語句中的舊跟猜面板（約 1219-1286 行）替換為新設計
   - 移除第二個 return 語句中重複的新面板（約 1747-1895 行）

2. **UI 設計特點**
   - 草藥紋理背景（fg-texture）
   - 角落裝飾圖案（fg-corner-tr、fg-corner-bl）
   - 標題區域：置中標題「跟猜階段」+ 紅色底線裝飾
   - 猜牌資訊區塊：顯示猜牌者和猜測的顏色（「紅色藥草」等）
   - 玩家決策狀態列表：卡片樣式垂直排列
   - 決定按鈕區：「不跟」和「跟猜」按鈕

3. **顏色標籤命名**
   - 紅色藥草、黃色藥草、綠色藥草、藍色藥草

4. **玩家狀態顯示**
   - 猜牌者：金色邊框 + 「猜牌者」標籤 + 「發起猜測」狀態
   - 已跟猜玩家：「跟隨」標籤（綠色）
   - 已不跟玩家：「不跟」標籤（灰色）
   - 當前決定者：橙色邊框高亮 + 「決定中...」標籤 + 放大效果
   - 等待中玩家：半透明 + 「等待中」文字（灰色斜體）
