# 完成報告 0116

**工作單編號：** 0116

**工作單標題：** 延長 LocalStorage 過期時間

**完成日期：** 2026-01-25

---

## 一、工作摘要

將 LocalStorage 中房間資訊的過期時間從 5 分鐘延長至 2 小時，解決長時間遊戲後重整無法重連的問題。

---

## 二、問題描述

**原本行為：**
- 過期時間設為 5 分鐘
- 一場遊戲可能超過 5 分鐘
- 超過後重整頁面，資料被判定過期
- 無法自動重連

**修復後行為：**
- 過期時間延長至 2 小時
- 足夠覆蓋絕大多數遊戲時長
- 重整後可正常取得房間資訊進行重連

---

## 三、修改內容

### 3.1 修改過期時間

**檔案：** `frontend/src/utils/localStorage.js`

**修改前：**
```javascript
// 檢查是否過期（5 分鐘）
const EXPIRY_TIME = 5 * 60 * 1000;
```

**修改後：**
```javascript
// 工單 0116：檢查是否過期（2 小時）
// 原本 5 分鐘太短，一場遊戲可能持續 30-60 分鐘
const EXPIRY_TIME = 2 * 60 * 60 * 1000;
```

### 3.2 更新測試案例

**檔案：** `frontend/src/components/Lobby/Lobby.test.js`

更新「房間資訊過期時不應嘗試重連」測試，將過期時間從 6 分鐘改為 3 小時：

```javascript

```

## 七、備註

選擇 2 小時的理由：
1. 一場遊戲通常 30-60 分鐘
2. 預留緩衝時間給中途休息的情況
3. 2 小時足夠長，又不會無限期佔用 localStorage
4. 過期後會自動清除，不會造成問題
