# 工單 0069 完成報告

**日期：** 2026-01-25

**工單標題：** 「其中一種全部」問牌方式洩漏手牌資訊

**完成狀態：** 完成

## 問題描述

當問牌方式為「其中一種顏色全部」時，原本的邏輯會根據被問牌玩家是否需要選擇來洩漏資訊：
- 出現選擇介面 → 其他玩家知道被問者兩種顏色都有
- 沒有選擇介面 → 其他玩家知道被問者只有一種或都沒有

## 修復內容

### 後端修改

**檔案：** `backend/server.js`

1. 無論有幾種顏色，都觸發選擇流程
2. 新增 `availableColors` 欄位告訴前端哪些顏色可選
3. `waitingForColorChoice` 事件不再傳送顏色資訊給其他玩家
4. `processColorChoice` 支援 `'none'` 表示無牌可給

```javascript
// 無論有幾種顏色，都觸發選擇流程
return {
  success: true,
  requireColorChoice: true,
  availableColors: availableColors, // 只給被問牌玩家知道
  // ...
};
```

### 前端修改

**檔案：** `frontend/src/components/GameRoom/GameRoom.js`

1. 根據 `availableColors` 顯示不同介面：
   - 兩種顏色都有 → 兩個按鈕都可選
   - 只有一種顏色 → 一個可選，一個顯示灰色 (無)
   - 都沒有 → 顯示「確認（無牌可給）」按鈕

2. 其他玩家只看到「等待 XXX 選擇...」，看不到任何顏色資訊

## 介面設計

### 兩種顏色都有
```
請選擇要給哪種顏色的全部牌：
[紅色]  [藍色]
```

### 只有一種顏色
```
請確認給出你有的顏色：
[紅色]  [藍色 (無)]  ← 灰色不可點
```

### 都沒有
```
你沒有這兩種顏色的牌。
[確認（無牌可給）]
```

### 其他玩家視角
```
等待 XXX 選擇要給哪種顏色...
```
