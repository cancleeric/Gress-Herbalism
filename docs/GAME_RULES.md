# 遊戲規則完整說明

## 一、遊戲概述

這是一款3-4人玩的推理桌遊，玩家需要透過問牌和推理來猜測桌面上的兩張蓋牌顏色。

## 二、遊戲設置

### 2.1 牌組構成
- **總牌數**：14張
- **顏色分布**：
  - 紅色：2張
  - 黃色：3張
  - 綠色：4張
  - 藍色：5張

### 2.2 遊戲開始
1. 洗牌：將14張牌完全打亂
2. 抽蓋牌：從牌組中抽出2張牌，蓋著放在桌面中央（不顯示顏色）
3. 發牌：將剩餘的12張牌平均分配給所有玩家
   - 3人遊戲：每人4張
   - 4人遊戲：每人3張
   - 如果無法完全平均分配，多餘的牌隨機分配給玩家

### 2.3 玩家資訊
- 每個玩家只能看到自己的手牌
- 其他玩家的手牌不可見
- 桌面上的兩張蓋牌對所有玩家都不可見（直到遊戲結束或猜對時）

## 三、遊戲流程

### 3.1 回合順序
- 玩家按順序輪流進行回合
- 每回合玩家可以選擇執行「問牌」或「猜牌」其中一個動作
- 執行完動作後，輪到下一個玩家

### 3.2 回合動作選擇
- **正常情況**：玩家可以選擇「問牌」或「猜牌」
- **特殊情況**：如果只剩一個玩家仍在遊戲中，該玩家**必須執行猜牌動作**（不能選擇問牌）

### 3.3 跳過已退出玩家
- 如果某個玩家已退出遊戲（`isActive: false`），輪流時會自動跳過該玩家
- 繼續輪到下一個仍在遊戲中的玩家

## 四、問牌規則

### 4.1 問牌動作組成
問牌動作包含三個部分（**必須按順序選擇**）：
1. **選擇兩個顏色**：從紅、黃、綠、藍中選擇兩個**不同**的顏色
2. **選擇目標玩家**：選擇一個其他玩家作為目標
3. **選擇要牌方式**：從三種方式中選擇一種（**基於前面選定的兩個顏色**）

### 4.2 三種要牌方式

#### 方式一：兩個顏色各一張
- **說明**：從選定的兩個顏色中，**各要一張**（注意：是「各一張」，不是「各全部」）
- **執行邏輯**：
  - 如果目標玩家兩個顏色都有 → 各給一張（共2張）
  - 如果目標玩家只有其中一個顏色 → **只給該顏色一張**（共1張，**不是給該顏色的全部**）
  - 如果目標玩家兩個顏色都沒有 → 給「沒有」的回應（0張）

#### 方式二：其中一種顏色全部
- **說明**：從選定的兩個顏色中，選擇一個顏色，要該顏色的全部
- **執行邏輯**：
  - 如果目標玩家有該顏色 → 給該顏色的所有牌
  - 如果目標玩家沒有該顏色 → 給「沒有」的回應（0張）
- **注意**：如果目標玩家兩種顏色都有，可以選擇其中一種顏色要全部

#### 方式三：給一張要全部
- **說明**：從選定的兩個顏色中，給目標玩家其中一個顏色一張，要另一個顏色的全部
- **執行邏輯**：
  1. 先從發起玩家手牌中移除指定顏色一張，加入目標玩家手牌
  2. 如果目標玩家有另一個顏色 → 給該顏色的所有牌
  3. 如果目標玩家沒有另一個顏色 → 給「沒有」的回應（0張）
  4. **重要**：即使目標玩家沒有另一個顏色，發起玩家已經給出的牌不會收回

### 4.3 問牌重要規則 ⚠️

1. **必須誠實給牌**：
   - 目標玩家**不能拒絕給牌**（除非真的沒有牌）
   - 必須按照要求給牌，有多少給多少
   - 不能選擇性給牌或隱瞞

2. **有多少給多少**：
   - 如果要求的牌不足，給出所有符合條件的牌
   - 例如：要兩個顏色各一張，但只有一個顏色 → 只給該顏色一張

3. **「沒有」的回應**：
   - 如果目標玩家沒有要求的牌，系統會返回「沒有」的回應
   - 這個回應會記錄到遊戲歷史中
   - 手牌不發生變化（方式三除外，因為發起玩家已經給出牌）

4. **問牌結果**：
   - 問牌成功：發起玩家收到牌，目標玩家失去牌
   - 問牌「沒有」：手牌不變，但資訊被記錄（其他玩家可以從遊戲歷史中看到）

## 五、猜牌規則

### 5.1 猜牌動作
- 玩家選擇兩個顏色作為猜測（顏色可以重複，例如：兩個紅色）
- 提交猜測後，系統會比對猜測的顏色與蓋牌的顏色

### 5.2 猜牌時查看答案
- **重要**：當玩家執行猜牌動作時，**猜牌者可以查看正確答案**（蓋牌的實際顏色）
- 這讓玩家在提交猜測前或提交後可以看到正確答案
- 其他玩家在猜牌階段看不到答案

### 5.3 猜對處理
- **結果**：該玩家獲勝，遊戲結束
- **公布答案**：將蓋牌的實際顏色**公布給所有玩家**（所有玩家都能看到）
- **遊戲狀態**：
  - `winner`: 設為獲勝玩家ID
  - `gamePhase`: 設為 `'finished'`
- **顯示**：所有玩家都能看到兩張蓋牌的實際顏色

### 5.4 猜錯處理
- **結果**：該玩家退出遊戲（`isActive: false`）
- **蓋牌處理**：**將兩張蓋牌蓋回桌面**（保持隱藏狀態，不顯示顏色）
- **遊戲狀態**：
  - 如果還有其他玩家：繼續遊戲，輪到下一個玩家
  - **如果只剩一個玩家且猜錯**：遊戲結束，但**沒有獲勝者**（`winner: null`, `gamePhase: 'finished'`）
- **顯示**：蓋牌保持隱藏，所有玩家都看不到顏色

### 5.5 只剩一個玩家時的強制猜牌
- **規則**：如果只剩一個玩家仍在遊戲中，該玩家**必須執行猜牌動作**
- **UI 顯示**：只顯示「猜牌」按鈕，隱藏「問牌」按鈕
- **結果**：
  - 猜對 → 該玩家獲勝
  - 猜錯 → 遊戲結束，沒有獲勝者

## 六、遊戲結束條件

### 6.1 有獲勝者的情況
- 任何玩家猜對蓋牌顏色 → 該玩家獲勝，遊戲結束
- 公布正確答案給所有玩家

### 6.2 沒有獲勝者的情況
- **條件**：只剩一個玩家且該玩家猜錯
- **結果**：遊戲結束，但沒有獲勝者（`winner: null`）
- **顯示**：顯示「遊戲結束，沒有獲勝者」的訊息
- **蓋牌**：保持隱藏狀態，不顯示顏色

## 七、遊戲歷史記錄

### 7.1 記錄內容
- 所有問牌動作（包括「沒有」的回應）
- 所有猜牌動作（包括猜對和猜錯）
- 玩家退出遊戲的記錄
- 遊戲結束的記錄

### 7.2 歷史記錄用途
- 玩家可以從歷史記錄中推斷其他玩家的手牌資訊
- 幫助玩家做出更好的決策

## 八、勝利條件

### 8.1 獲勝方式
- **唯一方式**：正確猜出桌上兩張蓋牌的顏色
- 猜測的顏色必須與蓋牌的顏色完全匹配（不考慮順序）

### 8.2 猜測規則
- 必須選擇兩個顏色（可以重複）
- 顏色順序不影響結果（例如：猜 [red, blue] 和 [blue, red] 視為相同）
- 系統會自動比對猜測與蓋牌

## 九、特殊情況處理

### 9.1 手牌為空
- 如果玩家手牌為空，仍然可以執行問牌或猜牌動作
- 問牌時如果目標玩家沒有要求的牌，返回「沒有」

### 9.2 所有玩家都退出
- 理論上不會發生（因為只剩一個玩家時必須猜牌）
- 如果發生，遊戲結束，沒有獲勝者

### 9.3 遊戲階段
- `'waiting'`: 等待玩家加入（3-4人）
- `'playing'`: 遊戲進行中
- `'finished'`: 遊戲結束（有獲勝者或沒有獲勝者）

## 十、遊戲策略提示

### 10.1 問牌策略
- 利用問牌收集資訊，推斷其他玩家的手牌
- 注意遊戲歷史記錄，分析其他玩家的問牌模式
- 利用「沒有」的回應來排除某些可能性

### 10.2 猜牌時機
- 當有足夠資訊推斷蓋牌顏色時，可以選擇猜牌
- 注意：猜錯會退出遊戲，要謹慎選擇
- 只剩一個玩家時必須猜牌，沒有選擇

### 10.3 資訊管理
- 觀察其他玩家的問牌行為
- 從遊戲歷史中推斷資訊
- 保護自己的手牌資訊不被推斷
