## 問題描述

### 症狀
前端顯示「與伺服器斷線，請確認後端是否啟動」錯誤，但後端服務實際正在運行且健康檢查通過。

### 根本原因分析

經過代碼分析，發現以下問題：

| 優先級 | 問題 | 檔案位置 | 影響 |
|--------|------|----------|------|
| 🔴 P1 | 前端 IP 地址硬編碼 | `frontend/.env` | 不同網路環境無法連接 |
| 🔴 P1 | 演化論遊戲缺少重連事件處理 | `backend/server.js` | 演化論遊戲斷線後無法重連 |
| 🟠 P2 | 心跳超時參數不一致 | `socketService.js` / `server.js` | 頻繁誤斷線 |
| 🟡 P3 | 缺少連線診斷功能 | - | 難以排查連線問題 |

---

## 詳細問題說明

### 問題 1：前端 IP 地址硬編碼（P1 - 臨界）

**現狀**：
```
# frontend/.env
REACT_APP_SOCKET_URL=http://192.168.50.132:3001
```

**問題**：
- 硬編碼特定區域網路 IP 地址
- 更換網路環境（WiFi、VPN、行動網路）後無法連接
- 其他開發者無法直接使用

**解決方案**：
- 改用 `localhost` 作為開發環境預設值
- 提供動態 IP 偵測機制作為備選

### 問題 2：演化論遊戲缺少重連事件處理（P1 - 臨界）

**現狀**：
```javascript
// frontend/src/services/reconnectionHandler.js 182-185
this.socket.emit('evo:reconnect', {
  roomId: this.roomId,
  playerId: this.playerId,
});

// backend/server.js - 無對應的 'evo:reconnect' 事件處理
```

**問題**：
- 前端發送 `evo:reconnect` 事件
- 後端未註冊該事件處理器
- 演化論遊戲斷線後無法正確重連

**解決方案**：
- 在後端添加 `evo:reconnect` 事件處理

### 問題 3：心跳超時參數不一致（P2 - 高）

**現狀**：
```javascript
// backend/server.js 336-337
pingTimeout: 30000,     // 30 秒
pingInterval: 25000,

// frontend/src/services/socketService.js 30-31
pingInterval: 25000,
pingTimeout: 20000,     // 20 秒（比後端短 10 秒）
```

**問題**：
- 前端 pingTimeout 比後端短 10 秒
- 網路延遲介於 20-30 秒時，前端誤判為斷線

**解決方案**：
- 統一前後端心跳超時參數

### 問題 4：缺少連線診斷功能（P3 - 中）

**現狀**：
- 無法快速診斷連線問題原因
- 排查困難

**解決方案**：
- 添加連線診斷工具
