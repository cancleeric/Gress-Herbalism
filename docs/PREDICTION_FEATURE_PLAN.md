# 預測功能 計畫書

**專案名稱：** Gress 推理桌遊 - 預測功能
**日期：** 2026-01-25
**版本：** 1.0

---

## 一、功能概述

「預測」是一個新的遊戲動作，讓玩家在問牌後可以選擇預測蓋牌中包含的顏色，增加遊戲的策略深度與趣味性。

---

## 二、遊戲規則

### 1. 預測時機

| 時機 | 說明 |
|------|------|
| 觸發條件 | 玩家完成「問牌」動作後 |
| 顯示對象 | 僅當前回合的玩家 |
| 是否強制 | 非強制，可選擇跳過 |

### 2. 預測流程

```
┌─────────────────────────────────────────────────────────┐
│                     預測流程圖                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   問牌完成                                               │
│       │                                                 │
│       ▼                                                 │
│   ┌─────────────────────────┐                           │
│   │  顯示預測選項            │                           │
│   │                         │                           │
│   │  [紅] [黃] [綠] [藍]    │  ← 四個顏色按鈕           │
│   │                         │                           │
│   │       [跳過]            │  ← 不預測                 │
│   └─────────────────────────┘                           │
│       │                                                 │
│       ├─── 選擇顏色 ───┐                                │
│       │                │                                │
│       ▼                ▼                                │
│   記錄預測          不記錄                               │
│       │                │                                │
│       └───────┬────────┘                                │
│               │                                         │
│               ▼                                         │
│         輪到下一位玩家                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3. 預測內容

- 從四種顏色（紅、黃、綠、藍）中選擇**一種**
- 預測「蓋牌中是否包含該顏色」

### 4. 預測公開性

| 項目 | 公開性 |
|------|--------|
| 誰進行了預測 | ✓ 公開 |
| 預測的顏色 | ✓ 公開 |
| 預測結果（對/錯） | 答案揭曉時公開 |

**遊戲紀錄顯示範例：**
```
小明 問牌後預測：[紅色]
小華 問牌後預測：[綠色]
小王 選擇不預測
```

### 5. 計分規則

當答案揭曉（有人猜牌成功）時，結算所有預測：

| 預測結果 | 分數變化 |
|---------|---------|
| 預測的顏色**在**蓋牌中 | **+1 分** |
| 預測的顏色**不在**蓋牌中 | **-1 分** |
| 未預測 | 0 分 |

**計分範例：**

假設蓋牌為 [紅色, 藍色]：

| 玩家 | 預測 | 結果 | 分數 |
|------|------|------|------|
| 小明 | 紅色 | ✓ 正確 | +1 |
| 小華 | 綠色 | ✗ 錯誤 | -1 |
| 小王 | 未預測 | - | 0 |
| 小李 | 藍色 | ✓ 正確 | +1 |

### 6. 結算時機

預測結果在以下情況結算：
- 有玩家猜牌**成功**時
- 該局以其他方式結束時

### 7. 預測記錄

每局遊戲中，玩家可以進行多次預測（每次問牌後一次）。所有預測記錄保留至該局結束。

## 四、技術規格

### 1. 遊戲狀態擴充

```javascript
// 遊戲狀態
const gameState = {
  // ... 現有欄位

  // 遊戲階段新增
  phase: 'predicting',  // 新增預測階段

  // 該局所有預測記錄
  predictions: [
    {
      playerId: 'player1',
      playerName: '小明',
      color: 'red',
      round: 3,
      timestamp: 1706169600000,
      isCorrect: null,  // 答案揭曉後填入
    }
  ],
};
```

### 2. 遊戲階段流程

```
WAITING → PLAYING → QUESTIONING → PREDICTING → PLAYING → ...
                                      ↓
                                  (換下一位玩家)
```

### 3. Socket 事件

```javascript
// 前端 → 後端
'submitPrediction'   // { color: 'red' } 提交預測
'skipPrediction'     // {} 跳過預測

// 後端 → 前端
'showPredictionOptions'  // {} 顯示預測選項（給當前玩家）
'predictionMade'         // { playerId, playerName, color } 廣播預測（給所有玩家）
'predictionSkipped'      // { playerId, playerName } 廣播跳過（給所有玩家）
'predictionsSettled'     // { results: [...] } 結算所有預測
```

### 4. 常數定義

```javascript
// shared/constants.js
const GAME_PHASES = {
  WAITING: 'waiting',
  PLAYING: 'playing',
  QUESTIONING: 'questioning',
  PREDICTING: 'predicting',    // 新增
  FOLLOW_GUESSING: 'followGuessing',
  ROUND_END: 'roundEnd',
  FINISHED: 'finished',
};

const PREDICTION_SCORE = {
  CORRECT: 1,   // 預測正確 +1
  WRONG: -1,    // 預測錯誤 -1
};
```
