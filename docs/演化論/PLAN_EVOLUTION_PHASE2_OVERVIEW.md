4. **品質保證**：斷線重連、行動裝置、效能優化

## 二、架構設計原則

### 2.1 可擴展性設計

為支援未來擴充包（如《飛翔》、《時間線》等），採用以下架構原則：

```
┌─────────────────────────────────────────────────────────────────┐
│                        擴充包系統架構                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────┐                                            │
│  │  Expansion      │  ← 擴充包註冊中心                          │
│  │  Registry       │     管理所有已啟用的擴充包                  │
│  └────────┬────────┘                                            │
│           │                                                      │
│  ┌────────┴────────┬────────────────┬────────────────┐         │
│  │                 │                │                │         │
│  ▼                 ▼                ▼                ▼         │
│  ┌─────────┐  ┌─────────┐    ┌─────────┐    ┌─────────┐       │
│  │ 基礎版  │  │  飛翔   │    │ 時間線  │    │ 大陸    │       │
│  │ Origin  │  │ Flight  │    │Timeline │    │Continents│       │
│  └─────────┘  └─────────┘    └─────────┘    └─────────┘       │
│       │            │              │              │              │
│       ▼            ▼              ▼              ▼              │
│  ┌──────────────────────────────────────────────────────┐      │
│  │              Trait / Card Definition                  │      │
│  │              性狀與卡牌定義層                          │      │
│  └──────────────────────────────────────────────────────┘      │
│                              │                                   │
│                              ▼                                   │
│  ┌──────────────────────────────────────────────────────┐      │
│  │              Rule Engine (規則引擎)                   │      │
│  │              處理所有性狀效果和互動                    │      │
│  └──────────────────────────────────────────────────────┘      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 模組化設計

```
┌─────────────────────────────────────────────────────────────────┐
│                         模組依賴關係                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  shared/                                                         │
│  └── constants/                                                  │
│      ├── evolution.js          ← 基礎常數（不直接修改）          │
│      └── expansions/           ← 擴充包常數（新增）              │
│          ├── index.js          ← 擴充包註冊                     │
│          ├── flight.js         ← 飛翔擴充                       │
│          └── timeline.js       ← 時間線擴充                     │
│                                                                  │
│  backend/logic/evolution/                                        │
│  ├── core/                     ← 核心邏輯（穩定）                │
│  │   ├── gameLogic.js                                           │
│  │   ├── cardLogic.js                                           │
│  │   ├── creatureLogic.js                                       │
│  │   ├── feedingLogic.js                                        │
│  │   └── phaseLogic.js                                          │
│  │                                                               │
│  ├── traits/                   ← 性狀處理器（可擴展）            │
│  │   ├── traitRegistry.js      ← 性狀註冊中心                   │
│  │   ├── baseTraits.js         ← 基礎版性狀                     │
│  │   └── expansions/           ← 擴充包性狀                     │
│  │                                                               │
│  └── rules/                    ← 規則引擎（可擴展）              │
│      ├── ruleEngine.js         ← 規則引擎核心                   │
│      ├── baseRules.js          ← 基礎規則                       │
│      └── expansions/           ← 擴充規則                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
