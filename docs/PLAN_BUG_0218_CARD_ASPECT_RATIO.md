# BUG 修復計畫書 — 顏色組合牌顯示比例問題

## 一、問題描述

**BUG 編號**：BUG-0218
**建立日期**：2026-01-31
**嚴重程度**：中等（UI 顯示不完整）

### 1.1 問題現象

顏色組合牌圖片被裁切成橫向長方形，只能看到圖片的一部分，無法看到完整的卡片內容。

### 1.2 預期行為

六張卡片應顯示為直立長方形，完整呈現卡片圖片內容。

### 1.3 實際行為

卡片以橫向比例顯示，圖片被 `object-fit: cover` 裁切，遺失上下部分。

## 二、問題分析

### 2.1 根本原因

CSS 中 `.playing-inquiry-grid` 設定了固定高度 `height: 320px`，並使用 3x2 網格排列。每張卡片的高度被限制，導致橫向比例。

### 2.2 原始圖片比例

用戶提供的卡片圖片為**直立長方形**（約 3:4 或類似比例），但目前的 CSS 將其強制顯示為橫向。

### 2.3 相關 CSS 樣式

```css
/* 目前的設定 */
.playing-inquiry-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
  height: 320px;  /* 固定高度導致卡片被壓縮 */
}
```

## 三、修復計畫

### 3.1 修改方向

將卡片從橫向改為直立長方形：
- 移除固定高度限制
- 調整卡片尺寸為直立比例（如 80px x 110px 或類似）
- 使用 `object-fit: contain` 或調整容器比例

### 3.2 實施步驟

#### 步驟一：修改 .playing-inquiry-grid

```css
.playing-inquiry-grid {
  display: grid;
  grid-template-columns: repeat(6, 1fr);  /* 改為一排六張 */
  /* 或保持 3x2 但調整高度 */
  gap: 12px;
  /* 移除固定高度 */
}
```

#### 步驟二：修改 .playing-inquiry-card

```css
.playing-inquiry-card {
  width: 80px;
  height: 110px;  /* 直立比例 */
  /* 或使用 aspect-ratio */
  aspect-ratio: 3/4;
}
```

#### 步驟三：調整響應式設計

確保在不同螢幕尺寸下卡片仍能正確顯示。

## 五、預期成果

六張顏色組合牌以直立長方形顯示，完整呈現卡片圖片，包含所有視覺元素（中藥材圖案、角落的顏色圖騰等）。
