# 遊戲規則快速參考

> 完整規則請參考：[遊戲規則完整說明](./GAME_RULES.md)

## 核心規則要點

### ⚠️ 問牌規則（必須遵守）

1. **必須誠實給牌**：目標玩家不能拒絕給牌（除非真的沒有牌）
2. **有多少給多少**：如果要求的牌不足，給出所有符合條件的牌
3. **三種要牌方式**：
   - **方式1**：兩個顏色各一張 → 如果兩個顏色都有，各給一張（共2張）；如果只有一個顏色，只給該顏色一張（共1張）；如果都沒有，給「沒有」
   - **方式2**：其中一種顏色全部 → 有就給全部，沒有就「沒有」
   - **方式3**：給一張要全部 → 先給牌，然後要全部（沒有也要「沒有」，但已給出的牌不收回）

### ⚠️ 猜牌規則（必須遵守）

1. **查看答案**：猜牌時，猜牌者可以查看正確答案（蓋牌的實際顏色）
2. **猜對**：公布答案給所有玩家，該玩家獲勝
3. **猜錯**：蓋牌蓋回桌面（保持隱藏），該玩家退出
4. **只剩一個玩家**：必須執行猜牌動作（不能選擇問牌）
5. **最後一人猜錯**：遊戲結束，沒有獲勝者

## 遊戲流程圖

```
遊戲開始
  ↓
洗牌 → 抽2張蓋牌 → 發牌給玩家
  ↓
玩家輪流回合
  ↓
選擇動作：問牌 或 猜牌
  ↓
[問牌] → 選擇顏色 → 選擇目標 → 選擇方式 → 執行要牌 → 記錄結果
  ↓
[猜牌] → 查看答案 → 選擇顏色 → 驗證結果
  ↓
  猜對 → 公布答案 → 遊戲結束（有獲勝者）
  猜錯 → 蓋回桌面 → 玩家退出
         ↓
    只剩一個玩家？ → 是 → 必須猜牌
         ↓
    遊戲結束（有/無獲勝者）
```

## 關鍵數據結構

### 問牌動作結果
```javascript
{
  success: boolean,
  result: {
    cardsGiven: Card[],      // 發起玩家給出的牌（類型3）
    cardsReceived: Card[],    // 發起玩家收到的牌
    hasCards: boolean        // 目標玩家是否有牌（false = "沒有"）
  },
  message: string
}
```

### 猜牌動作結果
```javascript
{
  success: boolean,
  isCorrect: boolean,
  revealedCards: Card[] | null,  // 猜對時顯示的蓋牌
  message: string
}
```

### 遊戲結束條件
- `winner !== null` → 有獲勝者
- `winner === null && gamePhase === 'finished'` → 沒有獲勝者（最後一人猜錯）

## 常見問題

**Q: 問牌時可以拒絕給牌嗎？**  
A: 不可以。必須誠實給牌，有多少給多少。只有真的沒有牌時才返回「沒有」。

**Q: 猜牌時其他玩家能看到答案嗎？**  
A: 猜牌時只有猜牌者能看到答案。猜對後才會公布給所有玩家。

**Q: 只剩一個玩家時可以做什麼？**  
A: 只能猜牌，不能問牌。系統會自動隱藏問牌按鈕。

**Q: 如果最後一個玩家猜錯會怎樣？**  
A: 遊戲結束，但沒有獲勝者。蓋牌保持隱藏。

**Q: 方式1要兩個顏色各一張，但目標只有一個顏色怎麼辦？**  
A: **只給該顏色一張**（不是給該顏色的全部）。方式1是「各一張」，所以如果只有一個顏色，就只給一張。有多少給多少，不能拒絕。
