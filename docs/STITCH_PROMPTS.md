# Google Stitch 提示詞 - 本草 Herbalism

## 使用方式
1. 前往 **https://stitch.withgoogle.com/**
2. 登入 Google 帳號
3. 複製以下提示詞貼入
4. 生成後可導出到 Figma 或下載代碼

---

## 提示詞 1：登入頁面

```
設計一個名為「本草 Herbalism」的中國風卡牌遊戲登入頁面。

需求：
- 優雅簡潔的中國風設計，帶有淡淡的草藥/自然元素
- 配色：深綠色 (#2E7D32) 為主色，米白色 (#FFF8E1) 為背景，金色 (#FFB300) 為點綴
- 置中排版，包含：
  - 頂部遊戲 Logo/標題「本草 Herbalism」
  - 副標題：「3-4人線上推理桌遊」
  - Google 登入按鈕（主要）
  - 訪客登入按鈕（次要）
  - 底部簡短遊戲介紹
- 現代簡約風格，圓角設計
- 支援手機響應式
- 支援深色模式
```

## 提示詞 2：遊戲大廳

```
設計一個多人卡牌遊戲的遊戲大廳頁面。

需求：
- 頂部導航欄：用戶頭像、暱稱、分數顯示
- 導航標籤：房間、好友、排行榜、個人檔案
- 主要內容區域：
  - 房間列表，每個房間卡片顯示：房間名稱、人數（如「2/4」）、狀態（等待中/遊戲中）
  - 每個房間卡片有「加入」按鈕
- 右下角浮動按鈕「創建房間」
- 快速加入按鈕（隨機配對）
- 配色：綠色主色 (#2E7D32)，米白背景 (#FFF8E1)
- 中國風裝飾元素
- 乾淨的卡片式佈局
```

## 提示詞 3：遊戲房間（等待階段）

```
設計一個 3-4 人卡牌遊戲的房間等待畫面。

需求：
- 頂部顯示房間名稱和房間代碼
- 4 個玩家位置，以圓形或網格排列：
  - 每個位置顯示：頭像、名稱、準備狀態
  - 空位顯示「等待玩家加入...」
  - 已準備的玩家顯示綠色勾勾
- 中央區域：
  - 「開始遊戲」按鈕（僅房主可見，3人以上準備才可點擊）
  - 「準備」切換按鈕（非房主玩家用）
- 底部：「離開房間」按鈕
- 聊天或快速表情符號（選用）
- 配色：綠色 (#2E7D32)、米白 (#FFF8E1)、金色點綴
```

## 提示詞 4：遊戲主畫面

```
設計一個推理卡牌遊戲的主要遊戲畫面，包含以下元素：

佈局（從上到下）：
1. 頂部：所有玩家的資訊（名稱、分數、手牌數量）
2. 中央：
   - 兩張蓋著的牌放在中間（要猜測的蓋牌）
   - 當前玩家指示器和回合計時器
3. 底部：
   - 玩家的手牌（3-4張顏色牌：紅、黃、綠、藍）
   - 行動按鈕：「問牌」和「猜牌」

需求：
- 卡牌顏色要明顯區分：
  - 紅色：#E53935
  - 黃色：#FDD835
  - 綠色：#43A047
  - 藍色：#1E88E5
- 蓋牌有裝飾性的背面圖案
- 被選中的牌有高亮邊框
- 當前玩家區域要強調顯示
- 側邊有遊戲紀錄面板（手機版可收合）
- 中國草藥主題，帶有淡淡的裝飾
- 乾淨、易讀的介面
```

## 提示詞 5：問牌選擇面板

```
設計一個卡牌遊戲的問牌選擇彈窗/面板。

三步驟選擇流程：
1. 第一步：「選擇兩個顏色」- 顯示 6 張顏色組合卡片：
   - 紅+黃、紅+綠、紅+藍
   - 黃+綠、黃+藍、綠+藍
   - 每張卡片顯示兩個顏色條紋

2. 第二步：「選擇目標玩家」- 顯示其他玩家列表（含頭像）

3. 第三步：「選擇要牌方式」- 三個選項：
   - 「兩色各一張」
   - 「其中一種全部」
   - 「給一張要全部」

需求：
- 頂部有步驟指示器（1-2-3）
- 可返回上一步的按鈕
- 底部確認按鈕
- 清晰的視覺層次
- 配色與主遊戲畫面一致
```

## 提示詞 6：猜牌與結果畫面

```
設計一個卡牌遊戲的猜牌畫面和結果彈窗。

猜牌畫面：
- 提示文字：「猜測兩張蓋牌的顏色」
- 兩個卡片位置讓玩家選擇顏色
- 顏色選擇器：紅、黃、綠、藍 四個選項
- 可以選擇相同顏色兩次
- 「確認猜測」按鈕

結果彈窗：
- 大字顯示結果：「猜對了！🎉」或「猜錯了！❌」
- 揭曉實際的蓋牌顏色
- 分數變化顯示：
  - 玩家名稱：+3（猜牌者）
  - 玩家名稱：+1（跟猜正確）
  - 玩家名稱：-1（跟猜錯誤）
- 「下一局」或「返回大廳」按鈕
- 獲勝時有慶祝動畫

配色：綠色主色，米白背景
```

## 提示詞 7：排行榜頁面

```
設計一個卡牌遊戲的排行榜頁面。

需求：
- 標籤篩選：「總分」、「勝率」、「場次」
- 前三名玩家用獎牌標示（金、銀、銅）
- 排行榜列表顯示：
  - 排名數字
  - 玩家頭像
  - 玩家名稱
  - 分數/數據
  - 勝率百分比
- 如果當前用戶不在前十，底部高亮顯示自己的排名
- 下拉刷新功能
- 返回大廳按鈕
- 配色：綠色 (#2E7D32)，前三名用金色
```

## 提示詞 8：個人檔案頁面

```
設計一個卡牌遊戲的個人檔案頁面。

需求：
- 頂部大頭像和顯示名稱
- 編輯個人資料按鈕
- 統計數據卡片：
  - 遊戲場次：[數字]
  - 獲勝場次：[數字]
  - 勝率：[百分比]
  - 總得分：[數字]
  - 最高分：[數字]
- 近期對戰紀錄列表：
  - 日期、結果（勝/敗）、最終分數
- 成就區塊（選用）
- 返回大廳按鈕
- 配色：綠色主色，米白背景
```

## 提示詞 9：跟猜階段畫面

```
設計一個卡牌遊戲的跟猜階段畫面。

情境：有玩家猜牌後，其他玩家要決定是否跟猜。

需求：
- 頂部提示：「小明 猜測蓋牌為 [紅色] [藍色]」
- 顯示猜牌玩家選擇的兩個顏色
- 中央大字提示：「你要跟猜嗎？」
- 說明文字：「跟猜正確 +1分，跟猜錯誤 -1分」
- 兩個大按鈕：
  - 「跟猜」（綠色，確認按鈕風格）
  - 「不跟」（灰色，次要按鈕風格）
- 倒數計時器（如：剩餘 10 秒）
- 其他玩家的決定狀態（已決定/等待中）
- 配色：綠色主色，米白背景
```

---

## 提示詞 10：預測選擇畫面

```
設計一個卡牌遊戲的預測選擇畫面。

情境：玩家問牌後，可以選擇預測蓋牌包含什麼顏色。

需求：
- 標題：「問牌完成！」
- 提示文字：「是否要預測蓋牌中有哪個顏色？」
- 四個顏色按鈕橫排：
  - 紅色 (#E53935)
  - 黃色 (#FDD835)
  - 綠色 (#43A047)
  - 藍色 (#1E88E5)
- 選中的顏色有勾勾標記
- 說明文字：「預測對 +1分，預測錯 -1分（可不選）」
- 底部「結束回合」按鈕
- 簡潔明瞭的設計
- 配色：綠色主色，米白背景
```

## 設計系統參數

貼到 Stitch 的 Customization 設定：

```
主題：淺色（支援深色模式）
主色：#2E7D32（深綠色）
輔色：#FFB300（金色）
背景色：#FFF8E1（米白色）
表面色：#FFFFFF（白色）
文字色：#424242（深灰色）
圓角：12px
字體：Noto Sans TC（繁體中文）
```
