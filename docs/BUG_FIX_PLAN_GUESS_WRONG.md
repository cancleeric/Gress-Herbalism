# BUG 修復計畫書：猜牌結果顯示與流程修正

**建立日期**：2026-01-27

**問題描述**：猜錯時不應公布蓋牌答案，結果面板的按鈕應只顯示給猜牌者，跟猜者等待猜牌者操作。

---

## 一、問題分析

### 現狀（有 BUG）

| 場景 | 目前行為 | 正確行為 |
|------|---------|---------|
| 猜錯 + 遊戲繼續 | 顯示簡短 toast 3 秒消失，不顯示結果面板 | 顯示完整結果面板，不公布蓋牌，猜牌者按「繼續觀戰遊戲」 |
| 猜錯 + 全員退出 | 顯示結果面板，**公布蓋牌答案**，所有人看到「下一局」 | 顯示結果面板，**不公布蓋牌答案**，只有猜牌者看到「下一局」 |
| 猜對 | 顯示結果面板，公布蓋牌答案，所有人看到「下一局」 | 顯示結果面板，公布蓋牌答案，**只有猜牌者**看到「下一局」 |

### 根本原因

1. **後端** `server.js` `validateGuessResult`：不論猜對猜錯都回傳 `hiddenCards`，洩露蓋牌答案
2. **前端** `GameRoom.js`：結果面板永遠顯示蓋牌區塊、永遠顯示按鈕給所有玩家
3. **前端** `GameRoom.js`：猜錯且遊戲繼續時（`continueGame: true`）只顯示 3 秒 toast，不顯示完整結果面板
4. **缺少機制**：猜牌者點「繼續觀戰遊戲」後需通知其他玩家關閉結果面板

### 涉及的規則

- **規則 5.3**：猜對 → 公布答案給所有玩家 ✓
- **規則 5.5**：猜錯 → 蓋牌保持隱藏，不顯示顏色 ✗（目前違反）

---

## 二、實施計畫

### 工單 0171：後端 — 猜錯時不洩露蓋牌 + 新增結果確認事件

**修改檔案**：`backend/server.js`

1. **修改 `validateGuessResult`**：猜錯時回傳 `hiddenCards: null`（不洩露答案）
2. **修改 `guessResult` 事件發送**：猜錯時 `hiddenCards` 設為 `null`
3. **新增 `dismissGuessResult` socket 事件處理**：
   - 猜牌者發送此事件 → 後端廣播給房間所有玩家
   - 用於「繼續觀戰遊戲」場景（遊戲已在進行中，不需要 startNextRound）

### 工單 0172：前端 — 猜牌結果面板重構

**修改檔案**：`frontend/src/components/GameRoom/GameRoom.js`、`frontend/src/services/socketService.js`

1. **統一結果面板顯示**：不論猜對猜錯、不論是否繼續遊戲，都顯示完整結果面板（移除 `showBriefGuessResult` toast 邏輯）
2. **蓋牌區塊條件顯示**：只在 `isCorrect === true && hiddenCards !== null` 時顯示蓋牌答案
3. **按鈕只顯示給猜牌者**：比對 `guessingPlayerId === currentPlayerId`
4. **按鈕文字和行為**：
   - 猜對 → 「下一局」→ 發送 `startNextRound`
   - 猜錯 + 遊戲繼續 → 「繼續觀戰遊戲」→ 發送 `dismissGuessResult`
   - 猜錯 + 全員退出 → 「下一局」→ 發送 `startNextRound`
   - 遊戲結束（finished）→ 「離開房間」→ 所有人可見
5. **監聽 `dismissGuessResult` 事件**：收到後所有玩家關閉結果面板
6. **socketService.js**：新增 `dismissGuessResult` 發送和監聽函數
