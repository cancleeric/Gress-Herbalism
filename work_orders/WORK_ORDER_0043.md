# 工作單 0043

**日期：** 2026-01-24

**工作單標題：** Bug 修復 - 最後一個玩家強制猜牌邏輯

**工單主旨：** Bug 修復 - 修正最後一個玩家的遊戲結束處理邏輯

**內容：**

## 問題描述

當前遊戲邏輯在處理「只剩一個玩家」的情況時可能存在錯誤。

### 正確規則

1. 如果只剩下一個玩家還沒猜牌（其他玩家都猜錯退出），該玩家**不是直接獲勝**
2. 該玩家必須**強制猜牌**
3. 如果最後一個玩家也猜錯，遊戲結束，**沒有人獲勝**（`winner: null`）

### 當前問題

需要檢查並確認以下邏輯是否正確實作：
- 只剩一個玩家時，是否正確強制進入猜牌模式
- 最後一個玩家猜錯時，是否正確設置 `winner: null`
- 前端 UI 是否正確隱藏「問牌」按鈕

## 需要檢查的檔案

1. **後端遊戲邏輯**
   - `backend/services/gameService.js` - 遊戲狀態管理
   - `backend/services/actionProcessor.js` - 動作處理器
   - `backend/services/guessProcessor.js` - 猜牌處理器

2. **前端顯示邏輯**
   - `frontend/src/components/GameRoom/GameRoom.js` - 遊戲房間組件
   - 確認只剩一個玩家時是否正確顯示強制猜牌介面

## 修復內容

1. 確認後端正確判斷「只剩一個玩家」的情況
2. 確認強制猜牌邏輯正確運作
3. 確認最後一個玩家猜錯時，`winner` 設為 `null`
4. 確認前端正確隱藏問牌選項

## 驗收標準

- [ ] 最後一個玩家猜錯 → 遊戲結束，沒有獲勝者
- [ ] 前端正確顯示「遊戲結束，沒有獲勝者」的訊息
- [ ] 遊戲歷史正確記錄此情況
