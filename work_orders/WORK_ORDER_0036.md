# 工作單 0036

**日期：** 2026-01-23

**工作單標題：** 修復創建房間玩家數量驗證錯誤

**工單主旨：** Bug 修復 - 創建房間時玩家數量驗證邏輯錯誤

**內容：**

## 問題描述

在遊戲大廳頁面，用戶選擇「3人」玩家數量並點擊「創建新房間」按鈕時，系統顯示錯誤訊息：

> 創建房間失敗：玩家數量必須在 3-4 人之間

但 3 人是有效的玩家數量（遊戲支援 3-4 人），不應該報錯。

## 重現步驟

1. 進入遊戲大廳頁面
2. 輸入玩家名稱
3. 在「玩家數量」下拉選單選擇「3人」
4. 點擊「創建新房間」按鈕
5. 出現錯誤訊息「創建房間失敗：玩家數量必須在 3-4 人之間」

## 預期行為

選擇 3 人或 4 人時，應該成功創建房間，不應該顯示錯誤。

## 可能原因

- 前端傳送的玩家數量格式可能有問題（例如傳送字串 "3人" 而非數字 3）
- 後端驗證邏輯可能有錯誤（例如使用 `>` 而非 `>=`）
- 前後端玩家數量欄位名稱不一致

## 工作內容

1. **檢查前端**
   - 檢查創建房間的 API 請求，確認傳送的玩家數量格式
   - 檢查下拉選單的 value 值

2. **檢查後端**
   - 檢查創建房間 API 的驗證邏輯
   - 確認玩家數量驗證條件是否正確

3. **修復問題**
   - 根據問題原因進行修復

4. **測試驗證**
   - 測試選擇 3 人創建房間
   - 測試選擇 4 人創建房間
   - 確認房間能正常創建

## 驗收標準

- [ ] 選擇 3 人時可以成功創建房間
- [ ] 選擇 4 人時可以成功創建房間
- [ ] 錯誤訊息只在真正無效的情況下顯示
