# 工作單 0010

**日期：** 2026-01-23

**工作單標題：** 建立遊戲服務 - 問牌動作處理

**工單主旨：** 服務層 - 實作問牌動作的處理邏輯

**內容：**

## 工作內容

1. **在 `frontend/src/services/gameService.js` 中新增函數**

2. **實作 `processQuestionAction(gameId, action)` 函數**
   - 處理問牌動作
   - 接收遊戲ID和問牌動作物件作為參數
   - 動作物件結構：
     ```javascript
     {
       type: 'question',
       playerId: string,
       targetPlayerId: string,
       colors: [color1, color2],
       questionType: 1 | 2 | 3
     }
     ```
   - 處理流程：
     1. 驗證當前玩家是否為動作發起者
     2. 驗證問牌動作是否有效（使用 `gameRules.js` 中的驗證函數）
     3. 根據問牌類型執行要牌邏輯（**重要：玩家必須誠實給牌，不能拒絕，有多少給多少**）：
        - **類型1**：從目標玩家要兩個顏色各一張（注意：是「各一張」，不是「各全部」）
          - 如果目標玩家兩個顏色都有，則各給一張（共2張）
          - 如果目標玩家只有其中一個顏色，則**只給該顏色一張**（共1張，**不是給該顏色的全部**）
          - 如果目標玩家兩個顏色都沒有，則給「沒有」的回應
        - **類型2**：從目標玩家要其中一個顏色的全部
          - 如果目標玩家有該兩種顏色其中之一，則挑選一種顏色並且給該顏色的所有牌
          - 如果目標玩家沒有該顏色，則給「沒有」的回應
        - **類型3**：給目標玩家其中一個顏色一張，要另一個顏色的全部
          - 先從發起玩家手牌中給目標玩家指定顏色一張
          - 如果目標玩家有另一個顏色，則給該顏色的所有牌
          - 如果目標玩家沒有另一個顏色，則給「沒有」的回應（但發起玩家已經給出的牌不會收回）
     4. 更新玩家手牌（根據實際給出的牌）
     5. 記錄動作到遊戲歷史（包括給牌結果和「沒有」的回應）
     6. 切換到下一個玩家
   - 返回處理結果物件：
     ```javascript
     {
       success: boolean,
       gameState: GameState,
       result: {
         cardsGiven: Card[],        // 發起玩家給出的牌（類型3時使用）
         cardsReceived: Card[],     // 發起玩家收到的牌（可能為空陣列）
         hasCards: boolean          // 目標玩家是否有牌可給（false 表示「沒有」）
       },
       message: string              // 結果訊息（例如：「收到2張牌」或「目標玩家沒有該顏色的牌」）
     }
     ```

3. **實作問牌類型的處理函數**
   - `handleQuestionType1()`: 處理兩個顏色各一張
     - 檢查目標玩家手牌中兩個顏色的牌
     - **重要**：方式1是「各一張」，所以如果只有一個顏色，只給一張（不是給該顏色的全部）
     - 如果兩個顏色都有，各給一張（共2張）
     - 如果只有一個顏色，只給該顏色一張（共1張）
     - 如果兩個顏色都沒有，返回 `hasCards: false`
   - `handleQuestionType2()`: 處理其中一種顏色全部
     - 檢查目標玩家手牌中指定顏色的所有牌
     - 如果有該顏色，給全部；如果沒有，返回 `hasCards: false`
   - `handleQuestionType3()`: 處理給一張要全部
     - 先從發起玩家手牌中移除指定顏色一張，加入目標玩家手牌
     - 檢查目標玩家是否有另一個顏色的牌
     - 如果有，給全部；如果沒有，返回 `hasCards: false`（但已給出的牌不收回）
   - 使用策略模式，方便未來擴展新的問牌類型

4. **處理目標玩家沒有牌的情況**
   - **重要規則：玩家必須誠實給牌，不能拒絕**
   - 如果目標玩家沒有要求的牌，返回 `hasCards: false` 和相應訊息
   - 更新遊戲歷史記錄「沒有」的回應
   - 不改變手牌（類型3除外，因為發起玩家已經給出牌）

5. **使用 JSDoc 註解**
   - 為所有函數添加完整的 JSDoc 註解
