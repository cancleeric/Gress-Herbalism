# 工作單 0072

**日期：** 2026-01-25

**工作單標題：** 給牌時顯示私密訊息給被要牌玩家

**工單主旨：** 功能開發 - 被要牌玩家可看到給出的牌與原因

---

## 功能概述

當玩家被要牌時，系統應顯示一則**私密訊息**給該玩家，說明：
1. 因為什麼問牌動作
2. 給出了什麼牌

此訊息**僅被要牌的玩家可見**，其他玩家看不到給出的具體牌。

## 功能需求

### 1. 訊息內容

被要牌的玩家會看到：

```
┌─────────────────────────────────────────┐
│  小明 向你問牌                           │
│                                         │
│  問牌方式：各一張                        │
│  選擇顏色：紅色、藍色                    │
│                                         │
│  你給出的牌：                            │
│  🔴 紅色 x1                             │
│  🔵 藍色 x1                             │
│                                         │
│              [確認]                      │
└─────────────────────────────────────────┘
```

### 2. 三種問牌方式的訊息範例

**方式一：各一張**
```
問牌方式：各一張
選擇顏色：紅色、藍色

你給出的牌：
🔴 紅色 x1
🔵 藍色 x1
共 2 張
```

**方式二：全部**
```
問牌方式：全部
選擇顏色：紅色、藍色

你給出的牌：
🔴 紅色 x2
🔵 藍色 x3
共 5 張
```

**方式三：其中一種全部**
```
問牌方式：其中一種全部
選擇顏色：紅色、藍色
你選擇給：紅色

你給出的牌：
🔴 紅色 x2
共 2 張
```

### 3. 資訊可見性

| 資訊 | 被要牌玩家 | 問牌玩家 | 其他玩家 |
|------|-----------|---------|---------|
| 問牌動作 | ✓ | ✓ | ✓ |
| 問牌方式 | ✓ | ✓ | ✗ |
| 選擇顏色 | ✓ | ✓ | ✗ |
| 給出的具體牌 | **✓ 私密** | ✗ | ✗ |
| 給出的張數 | ✓ | ✓ | ✓ |

### 4. 顯示時機與遊戲流程

**重要：此訊息為非阻塞式，遊戲不會等待被要牌玩家按確認才繼續。**

```
問牌流程：
┌─────────────────────────────────────────────────────┐
│  1. 問牌玩家選擇顏色和方式                            │
│  2. 問牌玩家選擇目標玩家                              │
│  3. 系統計算要給的牌                                  │
│  4. 牌轉移完成                                       │
│  5. 【同時】顯示私密訊息給被要牌玩家                   │
│  6. 【同時】問牌玩家進入預測/結束階段                  │
│  7. 被要牌玩家可隨時按確認關閉訊息                     │
└─────────────────────────────────────────────────────┘
```

### 5. 確認機制

| 項目 | 說明 |
|------|------|
| 確認按鈕 | 被要牌玩家必須按「確認」才能關閉訊息 |
| 遊戲流程 | **不阻塞**，遊戲繼續進行 |
| 訊息保留 | 訊息會持續顯示直到玩家按確認 |
| 目的 | 確保被要牌玩家知道自己給了什麼牌 |

**玩家體驗：**
- 被要牌玩家會看到彈窗訊息
- 可以先看清楚給出的牌
- 按確認後關閉彈窗
- 期間遊戲持續進行，不影響其他玩家
