# 工作單 0052

**日期：** 2026-01-24

**工作單標題：** 修復跟猜機制 - 按順位決定並公開顯示決定狀態

**工單主旨：** Bug 修復 - 跟猜應按順位輪流決定，且所有人可見決定狀態

**內容：**

## 問題描述

目前跟猜機制可能存在以下問題：
1. 跟猜決定沒有按照順位進行
2. 其他玩家看不到誰已經決定跟猜/不跟猜

## 預期行為

### 跟猜順序
1. 玩家 A 宣布猜牌
2. 從玩家 A 的**下一位**開始，按照順位輪流決定是否跟猜
3. 每位玩家必須等前一位玩家決定後，才能做出自己的決定
4. 所有活躍玩家都決定完畢後，才揭曉猜牌結果

### 決定狀態公開
- 所有玩家都可以即時看到：
  - 目前輪到誰決定
  - 每位玩家的決定狀態（等待中 / 跟猜 / 不跟猜）

## 範例流程（4人遊戲）

```
玩家順序：A → B → C → D

1. 玩家 B 宣布猜牌
2. 進入跟猜階段，順序為：C → D → A

   狀態顯示：
   - B: 猜牌者
   - C: ⏳ 決定中...
   - D: ⏸️ 等待中
   - A: ⏸️ 等待中

3. 玩家 C 決定「跟猜」

   狀態顯示：
   - B: 猜牌者
   - C: ✅ 跟猜
   - D: ⏳ 決定中...
   - A: ⏸️ 等待中

4. 玩家 D 決定「不跟猜」

   狀態顯示：
   - B: 猜牌者
   - C: ✅ 跟猜
   - D: ❌ 不跟猜
   - A: ⏳ 決定中...

5. 玩家 A 決定「跟猜」

   狀態顯示：
   - B: 猜牌者
   - C: ✅ 跟猜
   - D: ❌ 不跟猜
   - A: ✅ 跟猜

6. 所有人決定完畢，揭曉結果
```

## 受影響檔案

### 後端
- `backend/server.js`
  - 跟猜狀態管理：記錄當前輪到誰決定
  - 廣播事件：通知所有玩家目前的決定狀態
  - 順位邏輯：確保按順序進行

### 前端
- 跟猜 UI 組件
  - 顯示所有玩家的決定狀態
  - 只有輪到的玩家可以點擊決定按鈕
  - 即時更新狀態顯示

## 資料結構建議

```javascript
// 跟猜階段狀態
followGuessState: {
  guessingPlayerId: 'player_b',      // 猜牌者
  currentDeciderId: 'player_c',       // 目前輪到誰決定
  decisions: {
    'player_c': 'follow',             // 跟猜
    'player_d': 'pass',               // 不跟猜
    'player_a': null                  // 尚未決定
  },
  decisionOrder: ['player_c', 'player_d', 'player_a']  // 決定順序
}
```

## Socket 事件

### 伺服器 → 客戶端
- `followGuessUpdate`: 當有玩家做出決定時，廣播給所有人
  ```javascript
  {
    currentDeciderId: 'player_d',
    decisions: { 'player_c': 'follow', 'player_d': null, 'player_a': null }
  }
  ```

### 客戶端 → 伺服器
- `followGuessDecision`: 玩家提交決定
  ```javascript
  { gameId: '...', playerId: '...', decision: 'follow' | 'pass' }
  ```

- [ ] 每位玩家必須等前一位決定後才能決定
