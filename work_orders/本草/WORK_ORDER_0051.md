# 工作單 0051

**日期：** 2026-01-24

**工作單標題：** 修復「給一張要全部」要牌方式的顏色選擇權 Bug

**工單主旨：** Bug 修復 - 要牌方應有權選擇給哪種顏色的一張

**內容：**

## 問題描述

當使用問牌類型 3（給其中一種顏色一張，要另一種顏色全部）時：

- **要牌方（問牌的人）** 如果兩種顏色都有，應該可以**自己選擇**要給對方哪種顏色的一張
- 目前可能是系統隨機決定或沒有提供選擇介面

## 完整規則流程

### 問牌類型 3（給一張要全部）

1. 要牌方選兩種顏色 A 和 B
2. **要牌方選擇**給哪種顏色的一張（假設選 A）
3. 要牌方給被要牌方一張 A 色的牌
4. 被要牌方給要牌方**另一種顏色（B）的全部**

**關鍵：** 被要牌方要給的顏色是自動決定的 — 就是「要牌方沒給的那種顏色」的全部。

## 規則比較

| 問牌類型 | 選擇權 | 說明 |
|---------|--------|------|
| 類型 2（其中一種顏色全部） | **被要牌方** | 被要牌方選擇給哪種顏色的全部 |
| 類型 3（給一張要全部） | **要牌方** | 要牌方選擇給哪種顏色的一張，被要牌方自動給另一種的全部 |

## 預期行為

1. 要牌方選兩種顏色 A 和 B
2. **如果要牌方兩種顏色都有**：顯示介面讓要牌方選擇要給哪種顏色的一張
3. **如果要牌方只有一種顏色**：自動給有的那種
4. 要牌方給出一張後，被要牌方自動給「另一種顏色」的全部

## 受影響檔案

### 前端（主要）
- 問牌 UI 組件：需要新增讓要牌方選擇 `giveColor` 的介面
- `frontend/src/components/` - 相關問牌組件

### 後端
- `backend/server.js` - 確認 `giveColor` 參數正確處理，`getColor` 應為另一種顏色

## 測試案例

### 案例 1：要牌方兩種顏色都有
1. 玩家 A（要牌方）選紅、藍
2. 玩家 A 手上有紅牌和藍牌
3. **顯示選擇介面讓 A 選擇要給哪種顏色的一張**
4. A 選擇給紅色一張
5. A 給 B 一張紅牌
6. B 自動給 A 所有**藍牌**（另一種顏色）

### 案例 2：要牌方只有一種顏色
1. 玩家 A（要牌方）選紅、藍
2. 玩家 A 手上只有紅牌（沒有藍牌）
3. **不需顯示選擇介面**，自動給紅色一張
4. A 給 B 一張紅牌
5. B 自動給 A 所有**藍牌**

### 案例 3：被要牌方沒有另一種顏色
1. A 選紅、藍，給紅色一張
2. A 給 B 一張紅牌
3. B 手上沒有藍牌
4. B 不需給任何牌（沒有藍牌可給）

- [ ] 只有一種顏色時，系統自動處理
- [ ] 正確處理被要牌方沒有另一種顏色的情況
