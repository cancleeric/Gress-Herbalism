## 測試場景

### 場景 1：瀏覽器重新整理

1. 玩家 A 在遊戲中
2. 玩家 A 重新整理瀏覽器
3. 自動重連到遊戲
4. 執行問牌動作
5. 預測 UI 正確顯示

**預期結果：**
- 重連成功
- socketId 正確更新
- 遊戲流程正常

### 場景 2：網路斷線重連

1. 玩家 A 在遊戲中
2. 模擬網路斷線
3. 網路恢復後自動重連
4. 遊戲狀態正確恢復

**預期結果：**
- 自動重連成功
- 遊戲狀態同步

### 場景 3：重連時正在預測階段

1. 玩家 A 問牌完成，顯示預測 UI
2. 玩家 A 斷線
3. 玩家 A 重連
4. 預測 UI 重新顯示

**預期結果：**
- 重連後 postQuestionPhase 事件重新發送
- 預測 UI 正確顯示

### 場景 4：重連超時

1. 玩家 A 斷線
2. 等待超過 60 秒
3. 嘗試重連
4. 收到 reconnectFailed 訊息

**預期結果：**
- 重連失敗
- 顯示適當的錯誤訊息

### 場景 5：多分頁情境

1. 玩家 A 開啟兩個分頁
2. 都嘗試加入同一遊戲
3. 檢查行為是否正確

**預期結果：**
- 只有一個分頁有效
- 另一個分頁收到錯誤或被踢出

## 執行方式

1. 啟動本地服務
2. 開啟多個瀏覽器視窗模擬多玩家
3. 依序執行測試場景
4. 記錄測試結果
