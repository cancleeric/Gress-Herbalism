# 工作單 0023

**日期：** 2026-01-23

**工作單標題：** 整合遊戲房間組件 - 連接所有子組件

**工單主旨：** UI 組件 - 將所有子組件整合到遊戲房間組件中

**內容：**

## 工作內容

1. **在 `frontend/src/components/GameRoom/GameRoom.js` 中整合組件**

2. **匯入所有子組件**
   - 匯入 `GameBoard` 組件
   - 匯入 `PlayerHand` 組件
   - 匯入 `QuestionCard` 組件
   - 匯入 `GuessCard` 組件
   - 匯入 `GameStatus` 組件

3. **整合組件到佈局中**
   - 將 `GameBoard` 放在中央區域
   - 將 `PlayerHand` 放在底部區域
   - 將 `GameStatus` 放在側邊區域
   - 將 `QuestionCard` 和 `GuessCard` 作為 Modal 顯示

4. **實作操作按鈕**
   - 「問牌」按鈕：打開問牌介面
   - 「猜牌」按鈕：打開猜牌介面
   - 按鈕只在當前玩家回合時顯示
   - 按鈕只在遊戲進行中時顯示
   - **重要：如果只剩一個玩家，只顯示「猜牌」按鈕（隱藏「問牌」按鈕），因為該玩家必須執行猜牌動作**

5. **實作條件渲染**
   - 根據遊戲階段顯示不同的內容
   - 根據當前玩家顯示操作按鈕
   - 根據玩家狀態顯示不同的資訊
   - **如果只剩一個玩家，只顯示猜牌按鈕，不顯示問牌按鈕**
   - **猜牌時，顯示正確答案給猜牌者查看**
   - **猜對時，顯示正確答案給所有玩家**
   - **猜錯時，蓋牌保持隱藏狀態**

6. **實作遊戲流程控制**
   - 監聽遊戲狀態變化
   - 自動切換玩家回合
   - 處理遊戲結束的情況：
     - 有玩家獲勝：顯示獲勝者
     - **沒有獲勝者（只剩一個玩家且猜錯）：顯示「遊戲結束，沒有獲勝者」**
   - **檢查是否只剩一個玩家，如果是則強制顯示猜牌選項**

7. **連接 Redux Store**
   - 使用 `useSelector` 取得完整的遊戲狀態
   - 使用 `useDispatch` 發送動作
   - 確保所有子組件都能正確取得狀態

8. **優化樣式**
   - 調整各組件的佈局
   - 確保響應式設計
   - 優化視覺效果

9. **使用 JSDoc 註解**
   - 為組件添加 JSDoc 註解
