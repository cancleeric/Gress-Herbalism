# 工作單 0025

**日期：** 2026-01-23

**工作單標題：** 建立動作處理器 - 猜牌動作處理器

**工單主旨：** 工具函數 - 建立猜牌動作的處理器模組

**內容：**

## 工作內容

1. **建立 `frontend/src/utils/actionHandlers/guessAction.js` 檔案**

2. **實作猜牌動作處理器**
   - 建立 `handleGuessAction()` 函數
   - 接收遊戲狀態和猜牌動作作為參數
   - **重要：猜牌時，猜牌者可以查看正確答案（蓋牌的實際顏色）**
   - 返回處理後的遊戲狀態

3. **實作猜牌驗證邏輯**
   - 使用 `gameRules.js` 中的驗證函數
   - 比對猜測的顏色與蓋牌顏色
   - **檢查是否只剩一個玩家：如果只剩一個玩家，該玩家必須執行猜牌動作**

4. **實作猜對處理**
   - 設定獲勝者為猜牌玩家
   - 設定遊戲階段為 'finished'
   - **公布正確答案：將蓋牌的顏色顯示給所有玩家**
   - 記錄猜牌動作到遊戲歷史

5. **實作猜錯處理**
   - 標記玩家為非活躍狀態（退出遊戲）
   - **將兩張蓋牌蓋回桌面（保持隱藏狀態，不顯示顏色）**
   - 記錄猜牌動作到遊戲歷史
   - 檢查是否只剩一個玩家：
     - **如果只剩一個玩家且猜錯：遊戲結束，但沒有獲勝者（winner: null）**
     - 如果還有其他玩家：繼續遊戲，切換到下一個玩家
   - **如果只剩一個玩家：該玩家必須執行猜牌動作（不能選擇問牌）**

6. **實作遊戲結束邏輯**
   - 檢查遊戲結束條件：
     - 有玩家猜對 → 該玩家獲勝
     - 只剩一個玩家且猜錯 → 遊戲結束，沒有獲勝者
   - 更新遊戲狀態

7. **使用 JSDoc 註解**
   - 為所有函數添加完整的 JSDoc 註解

8. **添加擴展性標記**
   - 在可擴展的地方添加 `// TODO: 可擴展點` 註釋
