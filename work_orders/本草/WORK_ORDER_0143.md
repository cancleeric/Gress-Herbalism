# 工作單 0143

**日期：** 2026-01-26

**工作單標題：** 修復大廳頁面欄位寬度不一致問題

**工單主旨：** UI 修復 - 統一暱稱、創建房間、加入房間欄位與房間列表寬度

**優先級：** 中

**依賴工單：** 無

---

## 一、問題描述

### 現象
當大廳有可用房間時，「遊戲暱稱」、「創建新房間」、「輸入房間 ID」等欄位的寬度與下方「可用房間」表格寬度不一致。

### 根本原因
1. `.nickname-section` 設定 `max-width: 500px`
2. `.create-room-btn` 設定 `max-width: 500px`
3. `.join-room-section` 設定 `max-width: 500px`
4. `.room-table-container` 沒有 `max-width` 限制，會擴展到父容器全寬
5. 導致表格比上方欄位寬

---

## 二、修復方案

移除上方欄位的 `max-width` 限制，讓所有元素統一為全寬，與房間表格一致。

### 修改內容

```css
/* 移除 max-width 限制 */
.nickname-section {
  margin-bottom: 24px;
  width: 100%;
  /* 移除 max-width: 500px; */
}

.create-room-btn {
  /* ... 其他樣式保持不變 ... */
  width: 100%;
  /* 移除 max-width: 500px; */
}

.join-room-section {
  /* ... 其他樣式保持不變 ... */
  width: 100%;
  /* 移除 max-width: 500px; */
}

.error-message {
  /* 移除 max-width: 500px; */
}
```
