# BUG 修復計畫書 — Cloud Run 遊戲中斷線與重連失敗

## 日期
2026-01-28

## 問題描述
在 Cloud Run 雲端版本遊戲中途突然顯示「與伺服器斷線」，無法接收房間訊息。重新整理後也無法連回房間。

## 問題根因分析（多層面）

### 根因 1：Cloud Run 部署缺少關鍵設定
**嚴重度：致命**

`.git/hooks/pre-push` 部署腳本中，`gcloud run deploy` 命令**缺少以下關鍵參數**：
- `--timeout=3600`：Cloud Run 預設 WebSocket 連線超時 **5 分鐘**，超過即強制斷線
- `--session-affinity`：無 session affinity 導致重連時被路由到不同實例
- `--min-instances=1`：無最小實例數，冷啟動時舊實例的記憶體狀態全部丟失

**影響**：
- 遊戲超過 5 分鐘必定斷線
- 重連時可能被路由到不同實例，該實例無此遊戲的記憶體狀態
- 實例縮放時房間狀態丟失

### 根因 2：socketService.js 自動重連使用錯誤的 localStorage key
**嚴重度：高**

`socketService.js` 第 60-62 行讀取：
- `localStorage.getItem('lastRoomId')`
- `localStorage.getItem('lastPlayerId')`
- `localStorage.getItem('lastPlayerName')`

但 `Lobby.js` 使用 `saveCurrentRoom()` 存入 `gress_current_room`（JSON 格式）。

`createRoom()` 函數（socketService.js:268）只存了 `lastPlayerId` 和 `lastPlayerName`，**沒有存 `lastRoomId`**（因為此時還不知道 roomId）。而 `joinRoom()` 有存。

**影響**：創建房間的房主在自動重連時找不到 `lastRoomId`，重連邏輯不會觸發。

### 根因 3：心跳間隔不足以維持 Cloud Run 連線
**嚴重度：中**

前端心跳 30 秒、Socket.io pingInterval 25 秒，但 Cloud Run 的超時是在 HTTP 層面，這些應用層心跳不能保證 Cloud Run 不會超時。

## 修復方案

### 工單 0183：修復 Cloud Run 部署設定
修改 `.git/hooks/pre-push` 部署腳本，為後端加入：
- `--timeout=3600`（WebSocket 超時延長到 1 小時）
- `--session-affinity`（同一用戶連到同一實例）
- `--min-instances=1`（保持至少一個實例，避免冷啟動丟失狀態）

### 工單 0184：統一 localStorage 重連 key 並強化自動重連
1. 修改 `socketService.js` 的 `reconnect` 事件，改用 `getCurrentRoom()` 讀取正確的 `gress_current_room` key
2. `createRoom()` 函數增加 `onRoomCreated` 回調後補存 `lastRoomId`（或統一改用 `saveCurrentRoom`）
3. 統一 `leaveRoom()` 的清理邏輯，同時清除兩組 key

### 工單 0185：增強心跳機制與斷線提示
1. 前端心跳間隔從 30 秒縮短為 15 秒
2. 後端 `pingTimeout` 從 60 秒改為 30 秒（更快偵測斷線）
3. 斷線時前端顯示「正在重連...」而非直接顯示錯誤，等重連失敗後再顯示錯誤

## 涉及檔案

| 工單 | 檔案 | 修改類型 |
|------|------|---------|
| 0183 | `.git/hooks/pre-push` | 加入 Cloud Run 部署參數 |
| 0184 | `frontend/src/services/socketService.js` | 統一 localStorage key、強化自動重連 |
| 0185 | `frontend/src/services/socketService.js` | 心跳間隔縮短 |
| 0185 | `backend/server.js` | pingTimeout 調整 |

## 驗收標準
1. Cloud Run 後端部署包含 `--timeout=3600`、`--session-affinity`、`--min-instances=1`
2. 頁面重整後可成功重連到原房間
3. 創建房間的房主重連時也能正確找到房間資訊
4. 心跳間隔縮短，更快偵測斷線
5. 斷線後自動重連成功率提升
