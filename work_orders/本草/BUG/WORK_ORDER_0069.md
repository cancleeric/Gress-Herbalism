# 工作單 0069

**日期：** 2026-01-24

**工作單標題：** 「其中一種全部」問牌方式洩漏手牌資訊

**工單主旨：** BUG 修復 - 修正問牌選擇邏輯以避免洩漏玩家手牌資訊

**分類：** BUG

---

## 問題描述

當問牌方式為「其中一種顏色全部」時，目前的邏輯是：
- 被問牌玩家**只有一種顏色** → 自動給出該顏色，不需選擇
- 被問牌玩家**兩種顏色都有** → 顯示選擇介面讓玩家選擇

這個邏輯會洩漏資訊：**當其他玩家看到被問牌玩家正在選擇時，就知道該玩家兩種顏色都有。**

## 問題分析

### 資訊洩漏情境

1. A 玩家向 B 玩家問牌，選擇紅色和藍色，方式為「其中一種全部」
2. 如果 B 玩家**沒有出現選擇介面**，其他玩家就知道 B 只有紅色或藍色其中一種（或都沒有）
3. 如果 B 玩家**出現選擇介面**，其他玩家就知道 B 紅色和藍色都有

這違反了遊戲的推理公平性。

## 預期行為

**無論被問牌玩家有幾種顏色，都必須顯示選擇介面：**

| 情境 | 目前行為 | 正確行為 |
|------|---------|---------|
| 兩種顏色都有 | 顯示選擇介面 | 顯示選擇介面，可選任一種 |
| 只有一種顏色 | 自動給出，不選擇 | 顯示選擇介面，但只能選該種顏色 |
| 兩種都沒有 | 自動跳過 | 顯示選擇介面，顯示「無牌可給」後確認 |

### 介面設計建議

```
┌─────────────────────────────┐
│  請選擇要給的顏色            │
│                             │
│  [紅色 ✓]    [藍色 ✗]       │
│   (可選)      (無此顏色)     │
│                             │
│        [確認]               │
└─────────────────────────────┘
```

- 有該顏色：可以點選
- 無該顏色：顯示為不可選（灰色），但其他玩家看不到哪個是灰色的

### 其他玩家視角

其他玩家只會看到：
- 「B 玩家正在選擇要給的顏色...」

不會知道 B 玩家實際上有幾種顏色可選。

## 修復方向

1. 修改被問牌玩家的選擇邏輯
2. 無論有幾種顏色，都觸發選擇流程
3. 根據實際手牌情況，決定哪些選項可選
4. 確保其他玩家只看到「正在選擇」的狀態，看不到可選項目

## 受影響檔案

- 前端問牌/給牌組件
- 後端問牌邏輯處理
- Socket 事件廣播內容

## 驗收標準

- [ ] 「其中一種全部」方式時，被問牌玩家一定會進入選擇流程
- [ ] 只有一種顏色時，該顏色為唯一可選項
- [ ] 兩種都沒有時，顯示「無牌可給」並確認
- [ ] 其他玩家無法從選擇流程判斷被問牌玩家的手牌情況
- [ ] 選擇介面的等待時間對所有情況一致（避免從反應時間推斷）
