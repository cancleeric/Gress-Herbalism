# 工作單 0011

**日期：** 2026-01-23

**工作單標題：** 建立遊戲服務 - 猜牌動作處理

**工單主旨：** 服務層 - 實作猜牌動作的處理邏輯

**內容：**

## 工作內容

1. **在 `frontend/src/services/gameService.js` 中新增函數**

2. **實作 `processGuessAction(gameId, action)` 函數**
   - 處理猜牌動作
   - 接收遊戲ID和猜牌動作物件作為參數
   - 動作物件結構：
     ```javascript
     {
       type: 'guess',
       playerId: string,
       guessedColors: [color1, color2]
     }
     ```
   - 處理流程：
     1. 驗證當前玩家是否為動作發起者
     2. **檢查是否只剩一個玩家：如果只剩一個玩家，該玩家必須執行猜牌動作**
     3. 驗證猜牌是否有效（使用 `gameRules.js` 中的驗證函數）
     4. **猜牌時，猜牌者可以查看正確答案（蓋牌的實際顏色）**
     5. 比對猜測的顏色與蓋牌顏色
     6. 如果猜對：
        - 設定 winner 為該玩家
        - 設定 gamePhase 為 'finished'
        - **公布正確答案：將蓋牌的顏色顯示給所有玩家**
        - 記錄動作到遊戲歷史
     7. 如果猜錯：
        - 設定該玩家的 isActive 為 false（退出遊戲）
        - **將兩張蓋牌蓋回桌面（保持隱藏狀態，不顯示顏色）**
        - 記錄動作到遊戲歷史
        - 檢查是否只剩一個玩家：
          - **如果只剩一個玩家且猜錯：遊戲結束，但沒有獲勝者（winner: null, gamePhase: 'finished'）**
          - 如果還有其他玩家：繼續遊戲，切換到下一個玩家
   - 返回處理結果物件：
     ```javascript
     {
       success: boolean,
       gameState: GameState,
       isCorrect: boolean,
       revealedCards: Card[] | null,  // 猜對時顯示的蓋牌（公布答案），猜錯時為 null
       message: string
     }
     ```

3. **處理遊戲結束邏輯**
   - 當有玩家猜對時，結束遊戲，該玩家獲勝
   - **當只剩一個玩家且猜錯時，遊戲結束但沒有獲勝者（winner: null）**
   - **當只剩一個玩家時，該玩家必須執行猜牌動作（不能選擇問牌）**
   - 更新遊戲狀態為 'finished'

4. **使用 JSDoc 註解**
   - 為所有函數添加完整的 JSDoc 註解
