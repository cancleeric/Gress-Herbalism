# 工單 0129：猜牌選擇面板 UI 重新設計

## 目標
根據 Stitch 生成的設計稿，重新設計猜牌選擇面板（GuessCard），延續中國風草藥主題設計風格。

## 遊戲邏輯說明
- 玩家選擇猜測蓋牌的兩個顏色（可重複選擇同色）
- 猜錯會退出遊戲，這是高風險操作
- **不需要「查看答案」功能**：答案只有在猜對後才會揭示
- 選滿兩個顏色後才能確認猜牌

## 設計規格

### 整體佈局
- Modal 寬度：max-w-[500px]
- 米白色背景（#F5F1E6）+ 金色邊框（rgba 金色 30%）
- 圓角 xl，陰影效果
- 草藥紋理背景（herbal-pattern）
- 角落裝飾（中國風雲紋）

### 頂部區域
- 標題「猜牌」（粗體，3xl，font-display）
- 紅色底線裝飾（w-16，h-0.5，bg-primary）

### 警告區塊（醒目）
- 黃色背景（bg-yellow-100）
- 左側紅色邊條（border-l-4 border-primary）
- 警告圖示（Material Symbols: warning）
- 文字：「猜錯會退出遊戲！請謹慎選擇。」（紅色粗體）

### 顏色選擇區塊
- 標題：「選擇兩個顏色（可重複）」（置中）
- 四個顏色按鈕（grid 4 欄）：
  - 紅色 (#E53935) - game-red
  - 黃色 (#FDD835) - game-yellow
  - 綠色 (#43A047) - game-green
  - 藍色 (#1E88E5) - game-blue
- 每個按鈕為正方形（aspect-square）+ 圓角
- hover 效果：白色邊框
- active 效果：縮小（scale-95）
- 顏色名稱標籤在按鈕下方

### 已選顏色顯示區
- 虛線邊框容器（border-dashed border-gold/30）
- 標籤：「已選擇：」
- 已選顏色顯示為圓角標籤（pill/chip 樣式）
  - 背景：對應顏色 20% 透明度
  - 文字：對應顏色
  - 右側 ✕ 按鈕可移除
- 未選時顯示灰色斜體「尚未選擇」
- 選滿兩個後，顏色按鈕變灰禁用

### 底部按鈕區
- 背景：半透明白色（bg-white/30）
- 上方金色分隔線
- 取消按鈕：灰色邊框樣式（flex-1）
- 確認猜牌按鈕：紅色背景（flex-2）
  - 圖示：check_circle
  - 文字：確認猜牌
  - 陰影效果
  - 未選滿兩色時禁用

### 設計風格
- 配色：
  - primary: #e83b3e（紅色，危險/警告）
  - deep-green: #2f7f34（主色）
  - gold: #C5A059（點綴）
  - background-light: #F5F1E6（米白背景）
- 字體：Noto Serif TC（標題）、Noto Sans TC（內文）
- Material Symbols 圖示
- 圓角、陰影效果
- 草藥裝飾元素

## 技術實作

### 檔案變更
1. **GuessCard.js** - 更新 UI 結構
2. **GuessCard.css** - 新增/更新樣式

### 實作重點
