端對端測試完整的重新整理重連場景，涵蓋所有遊戲階段

## 內容

### 測試範圍
完整前後端系統的重連功能，模擬使用者在雲端環境操作

### 測試場景

#### TC-0194-01：等待階段重整
- 玩家在等待房間（gamePhase: waiting）時重整
- 預期：應能重連回房間，看到其他玩家

#### TC-0194-02：遊戲進行中重整（輪到自己）
- 輪到該玩家操作時重整
- 預期：重連後應顯示正確的遊戲畫面，輪到自己

#### TC-0194-03：遊戲進行中重整（非自己回合）
- 非自己回合時重整
- 預期：重連後應顯示其他玩家正在操作

#### TC-0194-04：問牌後預測階段重整
- 問牌完成、預測階段時重整
- 預期：重連後應恢復預測介面

#### TC-0194-05：跟猜階段重整
- 有人猜牌、等待跟猜決定時重整
- 預期：重連後應顯示跟猜選項

#### TC-0194-06：多人同時重整
- 多位玩家同時按 F5
- 預期：所有人都能重連回遊戲

#### TC-0194-07：重整後再重整
- 連續快速重整兩次
- 預期：最終仍能重連成功

### 雲端環境特殊因素

#### TC-0194-08：Cloud Run 特性分析
- Cloud Run 的 WebSocket 支援限制
- 負載均衡對 Socket.io 的影響
- 容器自動擴縮對長連線的影響

### 測試方式
完整路徑追蹤 + 雲端環境因素分析（不修改程式碼）
