# 工作單 0044

**日期：** 2026-01-24

**工作單標題：** Bug 修復 - 「其中一種顏色全部」要牌方式的選擇權

**工單主旨：** Bug 修復 - 修正要牌方式二的互動邏輯

**內容：**

## 問題描述

在「其中一種顏色全部」的要牌方式中，當被要牌的玩家兩種顏色都有時，目前的邏輯有誤。

### 正確規則

當選擇「其中一種顏色全部」時：
- 如果被要牌的玩家**只有其中一種顏色** → 直接給該顏色全部
- 如果被要牌的玩家**兩種顏色都有** → **被要牌的玩家可以自己選擇要給哪一種顏色的全部**
- 如果被要牌的玩家**兩種顏色都沒有** → 回應「沒有」

### 當前問題

目前實作是由要牌的玩家選擇要哪個顏色，但正確邏輯應該是：
- 要牌玩家只選擇「兩個顏色」和「其中一種顏色全部」這個方式
- **被要牌的玩家**在兩種顏色都有的情況下，有權選擇給哪一種

## 需要修改的功能

### 1. 後端邏輯修改

- `backend/services/askProcessor.js` 或相關問牌處理器
- 新增判斷：被要牌玩家是否兩種顏色都有
- 如果兩種都有，需要等待被要牌玩家的選擇

### 2. 前端互動修改

- 新增被要牌玩家的選擇介面
- 當被問到「其中一種顏色全部」且兩種顏色都有時：
  - 顯示選擇介面，讓被要牌玩家選擇要給哪種顏色
  - 其他玩家等待該玩家做出選擇

### 3. Socket.io 事件新增

- 新增事件：通知被要牌玩家需要選擇
- 新增事件：被要牌玩家提交選擇
- 新增事件：廣播選擇結果

## 流程說明

```
1. 玩家 A 選擇問牌方式：「其中一種顏色全部」（選了紅色和藍色）
2. 後端檢查玩家 B 的手牌
3. 情況一：玩家 B 只有紅色或只有藍色
   → 直接給該顏色全部（現有邏輯）
4. 情況二：玩家 B 紅色和藍色都有
   → 發送事件給玩家 B，請求選擇要給哪種顏色
   → 玩家 B 選擇後，給出該顏色全部
5. 情況三：玩家 B 兩種都沒有
   → 回應「沒有」（現有邏輯）
```

- [ ] 遊戲歷史正確記錄這個互動
