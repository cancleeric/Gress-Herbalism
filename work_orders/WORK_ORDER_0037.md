# 工作單 0037

**日期：** 2026-01-23

**工作單標題：** 修復房間加入與房間列表顯示問題

**工單主旨：** Bug 修復 - 房間系統相關功能異常

**內容：**

## 問題描述

房間系統存在兩個問題：

### 問題一：以房間 ID 加入房間失敗

在「加入房間」區塊輸入房間 ID 後點擊「加入房間」按鈕，功能出錯，無法成功加入房間。

### 問題二：可用房間列表不更新

創建房間成功後，「可用房間」區塊仍顯示「目前沒有可用的房間」，沒有顯示剛建立的房間。

## 重現步驟

### 問題一重現
1. 玩家 A 創建房間，取得房間 ID
2. 玩家 B 在「房間ID」輸入框輸入該 ID
3. 點擊「加入房間」按鈕
4. 出現錯誤 / 無法加入

### 問題二重現
1. 輸入玩家名稱
2. 選擇玩家數量
3. 點擊「創建新房間」
4. 房間創建成功
5. 「可用房間」區塊仍顯示「目前沒有可用的房間」

## 預期行為

### 問題一預期
- 輸入有效的房間 ID 後，應該成功加入該房間
- 輸入無效的房間 ID 時，應顯示適當的錯誤訊息

### 問題二預期
- 創建房間後，該房間應該出現在「可用房間」列表中
- 其他玩家的「可用房間」列表也應該即時更新

## 可能原因

### 問題一可能原因
- 加入房間 API 端點有錯誤
- 前端傳送的房間 ID 格式不正確
- WebSocket 房間加入事件未正確處理

### 問題二可能原因
- 創建房間後未觸發房間列表更新
- WebSocket 廣播房間列表事件未實作
- 前端未監聽房間列表更新事件

## 工作內容

1. **檢查加入房間功能**
   - 檢查前端加入房間的請求邏輯
   - 檢查後端加入房間的 API / WebSocket 事件處理
   - 確認房間 ID 的傳遞和驗證

2. **檢查房間列表更新功能**
   - 檢查創建房間後是否有廣播更新事件
   - 檢查前端是否有監聽房間列表更新
   - 確認房間列表的資料來源和更新機制

3. **修復問題**
   - 修復加入房間功能
   - 實作或修復房間列表即時更新

4. **測試驗證**
   - 測試以房間 ID 加入房間
   - 測試創建房間後列表更新
   - 測試多個客戶端的房間列表同步
