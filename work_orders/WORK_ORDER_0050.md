### 2. 整合測試場景

#### 場景 A：正常猜對流程
```
1. 玩家 A 猜牌
2. 玩家 B 選擇跟猜
3. 玩家 C 選擇不跟猜
4. 猜測正確
5. 驗證：A +3分，B +1分，C 不變
6. 當局結束，進入下一局
```

#### 場景 B：猜錯流程
```
1. 玩家 A 猜牌
2. 玩家 B 選擇跟猜
3. 玩家 C 選擇不跟猜
4. 猜測錯誤
5. 驗證：A 不扣分但退出，B -1分且退出，C 繼續
6. C 強制猜牌（只剩一人）
```

#### 場景 C：達到勝利分數
```
1. 玩家 A 目前 5 分
2. 玩家 A 猜對，+3 分 = 8 分
3. 驗證：遊戲結束，A 獲勝
```

#### 場景 D：0 分時跟猜錯誤
```
1. 玩家 B 目前 0 分
2. 玩家 B 跟猜錯誤
3. 驗證：B 分數維持 0（不會變負）
```

#### 場景 E：所有人都退出
```
1. 只剩玩家 A 和 B
2. 玩家 A 猜牌
3. 玩家 B 跟猜
4. 猜測錯誤
5. 驗證：A 和 B 都退出，當局結束（無人得分），進入下一局
```

#### 場景 F：多局累計分數
```
1. 第一局：A +3, B +1
2. 第二局：B +3, A 跟猜錯 -1
3. 驗證：A = 2分，B = 4分
```
