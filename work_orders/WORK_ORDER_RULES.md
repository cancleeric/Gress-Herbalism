# 工單處理規則

本文件定義工單執行的標準流程與規範。

---

## 工單處理流程

每張工單必須依照以下步驟執行：

### 1. 開發階段
- 閱讀並理解工單需求
- 依照工單內容進行開發
- 確保程式碼符合專案規範

### 2. 測試階段

完成開發後，**必須**執行以下測試：

#### 2.1 單元測試
```bash
cd frontend && npm test
cd backend && npm test
```

#### 2.2 整合測試
- 啟動前後端服務
- 驗證功能正常運作
- 確認無回歸問題

### 3. 版本控制

測試通過後：

| 動作 | 說明 |
|------|------|
| `git add` | 加入變更的檔案 |
| `git commit` | 提交變更，訊息格式見下方 |
| **不要** `git push` | 除非使用者明確要求上 GitHub |

#### Commit 訊息格式
```
feat/fix/docs: 工單標題 (工單編號)

- 變更項目 1
- 變更項目 2

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>
```

### 4. 撰寫報告

完成後撰寫對應的報告書：
- 檔案：`reports/REPORT_XXXX.md`
- 內容：完成項目、測試結果、變更檔案列表

---

## 重要提醒

- **不要主動推送到 GitHub**，等待使用者指示
- 測試失敗時，先修復問題再重新測試
- 每張工單獨立 commit，方便追蹤歷史

---

## 快速檢查清單

- [ ] 開發完成
- [ ] 單元測試通過
- [ ] 整合測試通過
- [ ] Git commit（不 push）
- [ ] 報告書撰寫完成
