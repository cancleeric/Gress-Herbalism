# 工作單 0021

**日期：** 2026-01-23

**工作單標題：** 建立猜牌介面組件

**工單主旨：** UI 組件 - 建立猜牌介面（GuessCard）組件

**內容：**

## 工作內容

1. **建立 `frontend/src/components/GuessCard/GuessCard.js` 檔案**

2. **建立組件基礎結構**
   - 建立函數式組件
   - 接收必要的 props（onSubmit, onCancel 等）
   - 使用 Modal 或 Dialog 形式顯示

3. **實作顏色選擇器**
   - 顯示四個顏色選項：紅色、黃色、綠色、藍色
   - 允許選擇兩個顏色（可以重複，例如兩個紅色）
   - 已選擇的顏色要有視覺反饋
   - 顯示已選擇的兩個顏色

4. **實作警告提示**
   - 顯示「猜錯會退出遊戲」的警告訊息
   - 警告訊息要醒目

5. **實作確認和取消按鈕**
   - 確認按鈕：提交猜牌動作
   - 取消按鈕：關閉猜牌介面

6. **實作表單驗證**
   - 驗證必須選擇兩個顏色
   - 顯示驗證錯誤訊息

7. **實作猜牌動作提交**
   - `handleSubmit()` 函數
   - 收集表單資料（猜測的兩個顏色）
   - 建立猜牌動作物件
   - 調用 `gameService.processGuessAction()` 處理猜牌動作
   - 更新 Redux store
   - 關閉猜牌介面

8. **實作猜牌結果顯示**
   - **猜牌時，顯示正確答案給猜牌者查看（在提交前或提交後顯示蓋牌的實際顏色）**
   - 顯示猜牌結果（正確或錯誤）
   - 如果猜對：
     - 顯示獲勝訊息
     - **公布正確答案給所有玩家（顯示蓋牌的實際顏色）**
   - 如果猜錯：
     - 顯示退出遊戲的訊息
     - **將蓋牌蓋回桌面（保持隱藏狀態，不顯示顏色）**
     - **如果只剩一個玩家且猜錯，顯示「遊戲結束，沒有獲勝者」的訊息**

9. **連接 Redux Store**
   - 使用 `useSelector` 取得遊戲狀態
   - 使用 `useDispatch` 發送猜牌動作

10. **建立樣式檔案**
    - 建立 `frontend/src/components/GuessCard/GuessCard.css`
    - 設定猜牌介面的佈局樣式
    - 設定警告訊息的樣式

11. **匯出組件**
    - 確保組件可以正確匯出和匯入

12. **使用 JSDoc 註解**
    - 為組件添加 JSDoc 註解
