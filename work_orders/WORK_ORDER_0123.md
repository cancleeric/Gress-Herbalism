# 工單 0123：遊戲房間等待階段 UI 重新設計

## 目標
根據 Stitch 生成的設計稿，重新設計遊戲房間等待階段（`GAME_PHASE_WAITING`）的 UI，使其符合中國風草藥主題設計風格。

## 設計規格

### 整體佈局
三欄式佈局（左-中-右），響應式設計（手機版改為單欄垂直排列）

### Header（頂部導航）
- 左側：
  - 品牌標誌「本草 Herbalism」
  - 狀態文字「等待房主開始」
- 右側：
  - 「離開房間」按鈕（紅色/次要樣式）

### 左欄（自己的玩家卡片 - Hero Card）
- 大型卡片框架帶邊框
- 玩家頭像（大）
- 名稱 + 「(我)」標示
- 顯示「0分」和「0張牌」
- 卡片下方：「開始遊戲」按鈕
  - 僅房主可見且可點擊
  - 人數不足時禁用（顯示「需要 X 人」）

### 中央區域（蓋牌展示）
- 標題「蓋牌」
- 兩張蓋著的牌（背面朝上）
- 牌面顯示問號或裝飾圖案
- 帶有淡淡的陰影效果
- 等待階段提示文字

### 右欄（玩家列表）
- 標題「玩家：2/4」（顯示當前人數/最大人數）
- 垂直排列的玩家卡片列表：
  - 每張卡片：頭像、名稱、分數、手牌數
  - 自己的卡片標示「(我)」
  - 房主標示「房主」
  - 空位顯示「等待中...」虛線框
- 底部「邀請好友」按鈕（複製房間連結）

### 設計風格
- 配色：
  - 主色：深綠色 `#2f7f34`
  - 點綴：金色 `#C5A059`
  - 背景：米白 `#F5F1E6`
  - 表面：白色 `#FFFFFF`
- 字體：
  - 標題：Noto Serif TC
  - 內文：Be Vietnam Pro / Noto Sans TC
- 圓角：12px
- 中國風草藥元素裝飾

## 技術實作

### 檔案變更
1. **GameRoom.js** - 新增等待階段專用 UI 區塊
2. **GameRoom.css** - 新增等待階段樣式
3. **GameRoom.test.js** - 更新測試

### 實作細節
1. 當 `gamePhase === GAME_PHASE_WAITING` 時，渲染新的等待階段 UI
2. 複用現有的：
   - `handleStartGame()` - 開始遊戲
   - `handleLeaveRoom()` - 離開房間
   - `getMyPlayer()` - 取得自己的資訊
3. 新增功能：
   - 複製房間連結功能（邀請好友）
   - 空位顯示邏輯（根據 maxPlayers）

### 響應式設計
- 桌面版（≥1024px）：三欄並排
- 平板版（768px-1023px）：左欄+中央並排，右欄下方
- 手機版（<768px）：單欄垂直堆疊

## 驗收標準
1. 等待階段顯示新設計的三欄佈局
2. Header 包含品牌名稱、狀態、離開按鈕
3. 左欄顯示自己的大型玩家卡片和開始按鈕
4. 中央顯示兩張蓋牌預覽
5. 右欄顯示玩家列表和邀請按鈕
6. 響應式設計正常運作
7. 配色和字體符合設計規格
8. 所有現有測試通過
9. 新增相關測試
