# 工作單 0078

**日期：** 2026-01-25

**工作單標題：** 「其中一種全部」要牌方式選擇邏輯修正

**工單主旨：** BUG 修復 - 避免暴露被要牌玩家的手牌資訊

**分類：** BUG

**相關工單：** 0069

---

## 問題描述

使用「其中一種全部」要牌方式時，需要讓被要牌玩家**一定要進入選擇流程**，以避免暴露手牌資訊。

## 目前問題

如果被要牌玩家只有一種顏色，可能會自動給牌而不顯示選擇介面，這會讓其他玩家知道該玩家沒有另一種顏色。

## 預期行為

**無論被要牌玩家有幾種顏色，都必須顯示選擇介面：**

### 選擇介面規則

| 情境 | 選項顯示 |
|------|---------|
| 兩種顏色都有 | 兩個選項都可選 |
| 只有一種顏色 | 只有該顏色可選（另一個顯示為不可選） |
| 兩種都沒有 | 顯示「無牌可給」 |

### 介面範例

**情境：問紅色和藍色，被要牌玩家只有紅色**

```
┌─────────────────────────────────────────┐
│   請選擇要給的顏色                        │
│                                         │
│   [紅色 ✓]     [藍色 ✗]                 │
│    可選擇       無此顏色                  │
│                                         │
│           [確認]                         │
└─────────────────────────────────────────┘
```

**重點：其他玩家只會看到「正在選擇中...」，不會知道哪個選項可選。**

### 其他玩家視角

```
┌─────────────────────────────────────────┐
│                                         │
│   小華 正在選擇要給的顏色...             │
│                                         │
└─────────────────────────────────────────┘
```

## 資訊隱藏原則

| 資訊 | 被要牌玩家 | 其他玩家 |
|------|-----------|---------|
| 選擇介面 | 看到完整選項 | 只看到「選擇中」 |
| 哪些顏色可選 | 知道 | **不知道** |
| 最終選擇結果 | 知道 | 只知道給了幾張 |

## 驗收標準

- [ ] 「其中一種全部」時，被要牌玩家**一定**進入選擇流程
- [ ] 選項中只有自己有的顏色可選
- [ ] 沒有的顏色顯示為不可選（灰色）
- [ ] 其他玩家無法看到哪些選項可選
- [ ] 其他玩家只看到「正在選擇中」的提示
- [ ] 兩種都沒有時顯示「無牌可給」
