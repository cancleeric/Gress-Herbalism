# 工作單建立規則

## 工作單格式規範

每張工作單必須包含以下欄位：

### 1. 編號
- 格式：從 `0001` 開始，依序遞增
- 範例：`0001`, `0002`, `0003`...

### 2. 日期
- 格式：建立工作單的日期
- 範例：`2026-01-23`

### 3. 工作單標題
- 格式：簡潔明確的標題，描述該工作單的主要任務
- 範例：`建立專案基礎架構`、`實作牌組初始化功能`

### 4. 工單主旨
- 格式：說明要做什麼部分，通常是一個功能模組或開發階段
- 範例：`專案初始化`、`牌組管理模組`、`問牌功能實作`

### 5. 內容
- 格式：詳細描述工作內容，包括：
  - 具體要實作的功能
  - 需要建立的檔案
  - 需要實作的邏輯
  - 相關的技術要求
  - 驗收標準（可選）

## 工作單檔案命名規則

- 格式：`WORK_ORDER_XXXX.md`（XXXX 為編號）
- 範例：`WORK_ORDER_0001.md`, `WORK_ORDER_0002.md`

## 工作單存放位置

- 所有工作單存放在 `work_orders/` 目錄下

## 工作流程

### 單一工單處理流程

每張工單完成時，依序執行：

1. **執行工作單**：按照工作單內容進行開發
2. **單元測試**：執行相關單元測試，覆蓋率須達 **75% 以上**
3. **撰寫報告書**：測試通過後，撰寫完成報告
4. **本地 Commit**：將完成的工作提交到本地版本控制

### 計畫書層級處理流程

當一份計畫書下的**所有工單**都完成後，依序執行：

1. **同步更新計畫書**：更新計畫書狀態，標記已完成的項目
2. **整合測試**：執行該計畫書範圍內的整合測試
3. **本地 Commit**：將計畫書更新與整合測試結果提交到本地版本控制

### 計畫書狀態更新內容

更新計畫書時須包含：
- 各工單完成狀態（✓ 標記）
- 實際完成日期
- 重要變更或偏差記錄
- 整合測試結果摘要

## 除錯流程

當單元測試失敗或發現程式錯誤時，須依照以下流程處理：

### 1. 撰寫分析報告

測試失敗或發現錯誤時，須另開**分析報告**，包含：

- **錯誤描述**：詳細描述錯誤現象
- **錯誤訊息**：完整的錯誤堆疊（stack trace）
- **重現步驟**：如何重現該錯誤
- **影響範圍**：哪些功能受影響
- **初步分析**：可能的原因推測
- **修復方案**：建議的修復方向

**分析報告命名規則**：
- 格式：`DEBUG_XXXX_描述.md`
- 範例：`DEBUG_0327_base擴充包載入失敗.md`
- 存放位置：`reports/debug/` 目錄下

### 2. 開立除錯工單

若除錯工作複雜或需要追蹤，須另開**除錯工單**：

- **工單編號**：使用 `XXXX-DBG` 格式（如 `0327-DBG`）
- **關聯工單**：標明原始工單編號
- **內容**：引用分析報告，詳述修復計畫
- **存放位置**：`work_orders/<遊戲>/除錯/` 目錄下
- **範例**：`work_orders/演化論/除錯/WORK_ORDER_0327-DBG.md`

### 3. 強化除錯輸出

除錯過程中應加入**除錯用日誌**，方便追蹤問題：

```javascript
// 除錯用日誌範例
const DEBUG = process.env.DEBUG_MODE === 'true';

function debugLog(context, message, data = null) {
  if (DEBUG) {
    console.log(`[DEBUG][${context}] ${message}`, data ? JSON.stringify(data, null, 2) : '');
  }
}

// 使用方式
debugLog('GameInitializer', 'Loading expansion', { expansionId });
debugLog('ExpansionLoader', 'Load result', result);
```

**除錯日誌規範**：
- 使用 `[DEBUG]` 前綴標識
- 包含上下文（模組/函數名稱）
- 輸出關鍵變數值
- 生產環境須關閉（透過環境變數控制）

### 4. 除錯完成後

除錯完成後須：
1. 更新分析報告，記錄**實際原因**與**解決方案**
2. 關閉除錯工單（若有開立）
3. 移除或保留除錯日誌（視需求決定）
4. 確保所有測試通過

## 報告書格式

每完成一份工作單後，需撰寫報告書，包含：
- 工作單編號
- 完成日期
- 完成內容摘要
- 遇到的問題與解決方案
- 測試結果
- 下一步計劃

## 報告書檔案命名規則

- 格式：`REPORT_XXXX.md`（XXXX 為工作單編號）
- 範例：`REPORT_0001.md`, `REPORT_0002.md`

## 報告書存放位置

- 所有報告書存放在 `reports/` 目錄下
